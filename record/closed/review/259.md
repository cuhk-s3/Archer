## Test Strategies

### Strategy 1: Fabs Select NSZ Propagation

- **Target**: Folds involving `fabs` and `select` or other conditional instructions.
- **Rationale**: The patch explicitly prevents copying the `nsz` flag to a `select` instruction when folding `fabs(select)`. We should test if there are other similar folds where `nsz` is incorrectly propagated to a `select` or other instructions that produce a `fabs` result, or if the absence of `nsz` on the `select` prevents valid optimizations that don't depend on the sign bit of zero.
- **Expected Issue**: The `nsz` flag on the select instruction might be incorrectly propagated or assumed to apply to the `fabs` result, leading to incorrect optimizations downstream that assume the sign bit of the `fabs` result can be flipped.

### Strategy 2: Copysign NSZ Semantics

- **Target**: Folds involving `copysign` and `nsz`.
- **Rationale**: The patch clarifies that `nsz` only allows non-deterministically flipping the sign bit of input operands. For `copysign`, the result sign should only depend on the second operand. We should test if `copysign` with `nsz` on the first operand incorrectly flips the result sign when the second operand is zero.
- **Expected Issue**: The `copysign` instruction might incorrectly assume the sign of the first operand (magnitude) can be flipped if it has the `nsz` flag, leading to incorrect results when the second operand (sign) is zero.

### Strategy 3: FNeg Fabs NSZ Interaction

- **Target**: Folds involving `fneg`, `fabs`, and `nsz`.
- **Rationale**: The patch ensures `fabs` always has an unset sign bit, even with `nsz`. We should test if `fneg(fabs(x))` with `nsz` is incorrectly folded to `fabs(x)` or `x`, assuming the sign bit doesn't matter, which would violate the new `nsz` semantics.
- **Expected Issue**: The `fneg` instruction might incorrectly be removed or folded away when applied to a `fabs` result with `nsz`, leading to an incorrect sign bit.

### Strategy 4: FCmp NSZ Zero Sign Bit

- **Target**: Folds involving `fcmp`, `nsz`, and zero values.
- **Rationale**: The new `nsz` semantics allow flipping the sign bit of input operands. We should test if `fcmp` with `nsz` correctly handles comparisons involving `0.0` and `-0.0`, especially when combined with other instructions like `select` or `minnum`/`maxnum`.
- **Expected Issue**: The `fcmp` instruction might incorrectly ignore the sign bit of zero when `nsz` is present on one of the operands but not the other, or when the comparison result is used in a context that cares about the sign bit.

### Strategy 5: FDiv NSZ Zero Divisor

- **Target**: Folds involving `fdiv`, `nsz`, and zero divisors.
- **Rationale**: The patch changes `nsz` to allow flipping input operands. For `fdiv`, if the divisor is `-0.0` and `nsz` is present, it could be treated as `0.0`. We should test if this leads to incorrect `copysign` folds or other issues.
- **Expected Issue**: The `fdiv` instruction might incorrectly fold `X / -0.0` to `copysign(inf, X)` when `nsz` is present, assuming the sign of `-0.0` can be flipped to `0.0`.
