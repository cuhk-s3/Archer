## Test Strategies

### Strategy 1: Test with EHPad Predecessor

- **Target**: Create a test case with a block containing an EHPad (like `catchpad` or `cleanuppad`) that has a conditional branch whose condition is known in a predecessor.
- **Rationale**: `SplitBlockPredecessors` returns `nullptr` if `BB->canSplitPredecessors()` is false. `canSplitPredecessors()` returns false if the block starts with an EHPad (e.g., `catchpad`, `cleanuppad`). If `foldCondBranchOnValueKnownInPredecessorImpl` tries to split a block that starts with an EHPad, `SplitBlockPredecessors` will return `nullptr`. Before the fix, this would cause a null pointer dereference when calling `EdgeBB->setName(...)`. The test should create a scenario where `foldCondBranchOnValueKnownInPredecessorImpl` attempts to thread a branch through a block starting with an EHPad.
- **Expected Issue**: A null pointer dereference crash in SimplifyCFG when `SplitBlockPredecessors` returns `nullptr`.
