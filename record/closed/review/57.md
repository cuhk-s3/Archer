## Test Strategies

### Strategy 1: Missing fcSNan in SrcDemandedMask for fptrunc

- **Target**: A test case that demands `fcQNan` from the result of an `fptrunc` instruction, where the source operand is known to be a signaling NaN (`fcSNan`).
- **Rationale**: The `simplifyDemandedUseFPClassFPTrunc` function initializes `SrcDemandedMask` with `DemandedMask`. However, `fptrunc` quiets signaling NaNs, meaning an `fcSNan` in the source becomes an `fcQNan` in the result. If the `DemandedMask` contains `fcQNan` but not `fcSNan`, the `SrcDemandedMask` will not contain `fcSNan`. This causes `SimplifyDemandedFPClass` on the source operand to assume that `fcSNan` is not demanded, which can lead to incorrect optimizations (e.g., replacing the source with `poison` if it is known to be `fcSNan`). To fix this, `SrcDemandedMask` should add `fcSNan` if `DemandedMask` contains `fcQNan`.
- **Expected Issue**: The compiler might incorrectly optimize the source of the fptrunc to poison or assume it is never a signaling NaN, leading to incorrect results or undefined behavior when the source is actually a signaling NaN.
