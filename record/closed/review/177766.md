## Test Strategies

### Strategy 1: Test exp with known non-NaN input and demanded NaN

- **Target**: An `exp` intrinsic where the input is known to be non-NaN (e.g., from a previous operation or `nnan` flag on the input) and the demanded mask includes NaN (e.g., an `is.fpclass` check for NaN).
- **Rationale**: The fix changes the return value of `SimplifyDemandedUseFPClass` for `exp` intrinsics from `getFPClassConstant` to `simplifyDemandedFPClassResult`. `simplifyDemandedFPClassResult` not only tries to fold to a constant but also infers fast-math flags (like `nnan`, `ninf`) based on the known input classes and demanded mask. If the input is known to never be NaN, `simplifyDemandedFPClassResult` might infer `nnan` and drop UB-implying attributes. We should test if this inference is correct and doesn't introduce miscompilations when the demanded mask includes NaN but the input is known not to be NaN.
- **Expected Issue**: The `exp` intrinsic might be incorrectly folded to a constant or have fast-math flags incorrectly inferred, leading to incorrect results or missing UB-implying attributes.

### Strategy 2: Test exp with known non-Inf input and demanded Inf

- **Target**: An `exp` intrinsic where the input is known to be non-Inf and the demanded mask includes Inf.
- **Rationale**: Similar to the NaN case, `simplifyDemandedFPClassResult` can infer `ninf` if the input is known to never be Infinity. We should test if this inference is correct when the demanded mask includes Infinity but the input is known not to be Infinity.
- **Expected Issue**: The `exp` intrinsic might be incorrectly folded to a constant or have fast-math flags incorrectly inferred, leading to incorrect results or missing UB-implying attributes.

### Strategy 3: Test exp with known positive input and demanded negative

- **Target**: An `exp` intrinsic where the demanded mask only includes negative classes (e.g., `fcNegative`).
- **Rationale**: `exp` always returns a positive value. If the demanded mask only asks for negative values, the previous code might have returned poison or a constant. The new code uses `simplifyDemandedFPClassResult`. We should test if this handles the case correctly where the demanded mask is disjoint from the possible return values of `exp`.
- **Expected Issue**: The `exp` intrinsic might be incorrectly folded to a constant or have fast-math flags incorrectly inferred, leading to incorrect results or missing UB-implying attributes.

### Strategy 4: Test exp with known zero input and demanded zero

- **Target**: An `exp` intrinsic where the input is known to be zero and the demanded mask includes zero or positive normal.
- **Rationale**: `exp(0)` is 1. If the input is known to be zero, `exp` should return 1. The new code uses `simplifyDemandedFPClassResult`. We should test if this correctly folds to 1 and infers flags.
- **Expected Issue**: The `exp` intrinsic might be incorrectly folded to a constant or have fast-math flags incorrectly inferred, leading to incorrect results or missing UB-implying attributes.

### Strategy 5: Test exp with known negative infinity input and demanded zero

- **Target**: An `exp` intrinsic where the input is known to be negative infinity and the demanded mask includes zero.
- **Rationale**: `exp(-inf)` is 0. If the input is known to be negative infinity, `exp` should return 0. The new code uses `simplifyDemandedFPClassResult`. We should test if this correctly folds to 0 and infers flags.
- **Expected Issue**: The `exp` intrinsic might be incorrectly folded to a constant or have fast-math flags incorrectly inferred, leading to incorrect results or missing UB-implying attributes.
