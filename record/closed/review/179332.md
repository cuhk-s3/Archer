## Test Strategies

### Strategy 1: Missing Profile Metadata on Select

- **Target**: A test case containing a `umax` intrinsic that can be expanded into a `select` instruction, within a function that has profile data (e.g., `!prof !1` where `!1 = !{!"function_entry_count", i64 100}`).
- **Rationale**: The patch changes `Builder.CreateSelect` to `Builder.CreateSelectWithUnknownProfile`. The original code created a select instruction without any profile metadata. The new code creates a select instruction and explicitly sets its profile metadata to "unknown" if the function has profile data. This is because PGO instrumentation does not support intrinsics, so the original `umax` intrinsic would not have profile data, but the expanded `select` instruction should have some profile data to avoid confusing downstream passes. If we test the original code with a function that has profile data, the generated `select` instruction will lack profile data.
- **Expected Issue**: The generated select instruction will not have profile metadata, which might cause downstream passes to make suboptimal decisions or fail assertions if they expect profile metadata to be preserved.
