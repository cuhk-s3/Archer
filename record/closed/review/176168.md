### Bug #1

**Analysis:**
The original IR returns poison when x == y because cttz(0, true) is poison, and the select condition is false, so it returns the poison value. The transformed IR returns 64 unconditionally. This is an invalid refinement because the original program has undefined behavior (returning poison), but the transformed program does not. Wait, returning poison is not UB, it's just a poison value. But if the original program returns poison, the transformed program can return any value, which is a valid refinement. Let me check alive2. Alive2 says "Transformation seems to be correct!". So it's not a bug.
