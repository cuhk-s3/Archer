### Strategy 1: Outer GEP has select with !prof metadata

- **Target**: A chained GEP where the inner GEP has a constant index and the outer GEP has a select index with `!prof` metadata.
- **Rationale**: The patch ensures that metadata from the select instruction is propagated to the new select instruction when folding a chained GEP. This strategy tests the case where the outer GEP has the select instruction.
- **Expected Issue**: Without the patch, the new select instruction would lose the `!prof` metadata.

- **Target**: A chained GEP where the select instruction has multiple uses.
- **Rationale**: The optimization requires `Select->hasOneUse()`. If it has multiple uses, the optimization should not trigger.
- **Expected Issue**: The optimization should not apply, and the original select instruction (with its metadata) should remain.
