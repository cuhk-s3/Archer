## Test Strategies

### Strategy 1: Non-dominating early exits

- **Target**: Create a loop with two early exits where neither dominates the other.
- **Rationale**: The patch requires that multiple early exits form a strict dominance chain. If we have two early exits that are independent (e.g., in different branches of an if-else statement), they won't dominate each other. The compiler should reject this.
- **Expected Issue**: The compiler should reject the loop for vectorization because the early exits do not form a dominance chain.

### Strategy 2: Last early exit not latch predecessor

- **Target**: Create a loop where the last early exit block is not the unique predecessor of the latch block.
- **Rationale**: The patch requires that the last early exit in the dominance chain is the unique predecessor of the latch block. If there's another block between the last early exit and the latch, or if the latch has multiple predecessors, it should be rejected.
- **Expected Issue**: The compiler should reject the loop for vectorization because the last early exit is not the unique predecessor of the latch.

### Strategy 3: Unmovable load in second early exit

- **Target**: Create a loop with multiple early exits where the condition for the second early exit involves a potentially faulting load that cannot be hoisted.
- **Rationale**: The patch checks if the condition load for all uncountable early exits can be moved. If a load in a subsequent early exit block cannot be safely moved (e.g., it might fault and isn't guaranteed to execute), vectorization should fail.
- **Expected Issue**: The compiler should reject the loop for vectorization because the load in the second early exit block cannot be moved.

### Strategy 4: Multiple uncountable early exits rejection in LoopVectorize

- **Target**: Create a valid loop with multiple uncountable early exits that passes legality checks but is rejected by LoopVectorize.cpp.
- **Rationale**: While LoopVectorizationLegality now accepts multiple early exits, LoopVectorize.cpp explicitly rejects them with "Auto-vectorization of loops with multiple uncountable early exits is not yet supported". We should test that this rejection happens correctly.
- **Expected Issue**: The compiler should reject the loop for vectorization because multiple uncountable early exits are not yet supported in LoopVectorize.cpp.
