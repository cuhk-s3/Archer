## Test Strategies

### Strategy 1: Pointer Induction with Exit User

- **Target**: A loop with a pointer induction variable that is used outside the loop (e.g., returned or stored after the loop).
- **Rationale**: The patch replaces extracts of the backedge value of a wide pointer induction with `ExitingIVValue`. Later, `optimizeInductionExitUsers` will process this `ExitingIVValue`. We need to ensure that `optimizeLatchExitInductionUser` correctly handles pointer inductions. Looking at `optimizeLatchExitInductionUser`, it has a branch for `ScalarTy->isPointerTy()`, which creates a `PtrAdd` with a negated step. We should test if this logic works correctly for pointer inductions that are used outside the loop.
- **Expected Issue**: The compiler will crash or generate incorrect code because `optimizeInductionExitUsers` expects `ExitingIVValue` to be used for pointer inductions, but it might not be able to handle the pointer type correctly when computing the end value or subtracting the step.
