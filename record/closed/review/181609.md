## Test Strategies

### Strategy 1: Memset Undef + Store

- **Target**: A function with a large memset of undef followed by an adjacent store of a specific value.
- **Rationale**: This directly tests the scenario described in the PR where a store is merged into a preceding memset of undef.
- **Expected Issue**: The pass merges the store into the undef memset, creating a single large memset of 0.

### Strategy 2: Store Undef + Store

- **Target**: A function with a store of undef followed by an adjacent store of a specific value.
- **Rationale**: This tests if the same logic applies when the first instruction is a store of undef instead of a memset.
- **Expected Issue**: The pass merges the second store into the first undef store, creating a memset of 0 covering both.

### Strategy 3: Store Undef + Memset

- **Target**: A function with a store of undef followed by an adjacent memset of a specific value.
- **Rationale**: This tests if the logic applies when the first instruction is a store of undef and the second is a memset of a specific value.
- **Expected Issue**: The pass merges the memset into the undef store, creating a single large memset of 0.

### Strategy 4: Memset Undef + Memset

- **Target**: A function with a memset of undef followed by an adjacent memset of a specific value.
- **Rationale**: This tests if the logic applies when both instructions are memsets, with the first being undef.
- **Expected Issue**: The pass merges the second memset into the first undef memset, creating a single large memset of 0.

### Strategy 5: Memset Undef + Store with Gap

- **Target**: A function with a memset of undef and a store of a specific value at an offset within the memset's range.
- **Rationale**: This tests if the pass merges a store into an undef memset even if there is a gap between them, as long as they are in the same range.
- **Expected Issue**: The pass merges the store into the undef memset, creating a single large memset of 0.
