## Test Strategies

### Strategy 1: Floating-point Induction Variable Live-Out

- **Target**: A loop with a floating-point induction variable that is used outside the loop.
- **Rationale**: FP induction variables often cannot have their exit values computed by SCEV. The vectorizer must extract the value from the vector. With tail folding, the last lane might be inactive. If the vectorizer extracts from the last lane instead of the last active lane, the live-out value will be incorrect.
- **Expected Issue**: The live-out value of the FP IV might be extracted from the last lane (which is inactive and contains an out-of-bounds value) instead of the last active lane, resulting in an incorrect value being used outside the loop.

### Strategy 2: Pointer Induction Variable Live-Out

- **Target**: A loop with a pointer induction variable that is returned or used after the loop.
- **Rationale**: Pointer IVs are common in C/C++ (e.g., `while (ptr < end) ptr++;`). If tail folding is applied and the pointer is used outside the loop, extracting from the wrong lane will yield an incorrect pointer value.
- **Expected Issue**: The live-out pointer might point beyond the intended end address if extracted from an inactive lane, leading to out-of-bounds memory accesses later in the program.

### Strategy 3: Non-unit Step Integer Induction Variable

- **Target**: A loop with an integer IV with a step > 1, where the IV is used outside the loop.
- **Rationale**: An integer IV with a step other than 1 (e.g., step = 3). If the trip count is not a multiple of the vectorization factor, the inactive lanes will contain values that would have been reached in out-of-bounds iterations.
- **Expected Issue**: The live-out value might be incorrect if the vectorizer assumes the exit value can be computed by simply taking the last lane's value and adding the step, which would include the step additions from inactive lanes.

### Strategy 4: Truncated Induction Variable Live-Out

- **Target**: A loop where an IV is truncated (e.g., `i64` to `i32`) and the truncated value is used outside the loop.
- **Rationale**: When an IV is truncated inside the loop and the truncated value is used outside, the vectorizer needs to ensure that the correct active lane of the truncated vector is extracted.
- **Expected Issue**: The extraction logic might fail to correctly identify the last active lane for the truncated value, or might extract from the un-truncated IV and truncate incorrectly, leading to a wrong live-out value.

### Strategy 5: Multiple Induction Variables with Outside Users

- **Target**: A loop with at least two induction variables (e.g., one counting up, one counting down), both of which are used outside the loop.
- **Rationale**: A loop might have multiple IVs (e.g., one incrementing, one decrementing). Testing multiple IVs ensures that the extraction logic correctly handles all of them, not just the primary canonical IV.
- **Expected Issue**: The vectorizer might correctly handle the primary IV using SCEV but fail to correctly extract the secondary IV from the last active lane, leading to an incorrect live-out for the secondary IV.
