- **Target**: Shl with 0 shift amount converted to other opcodes
- **Rationale**: The `isBinOpWithConstantInt` function allows `Pos == 0` for `Shl` if operand 1 is not a constant but operand 0 is. Ah, `if (Opcode == Instruction::Sub || Opcode == Instruction::Shl || Opcode == Instruction::AShr) return {nullptr, 0};` if operand 1 is not a constant. So `Pos` is always 1 for `Shl`. If `ToOpcode` is `Sub`, `getBinOpIdentity` returns 0. If `ToOpcode` is `And`, it returns -1. If `ToOpcode` is `Or`, it returns 0. If `ToOpcode` is `Xor`, it returns 0. If `ToOpcode` is `Add`, it returns 0. If `ToOpcode` is `Mul`, it returns 1. Wait, if `FromOpcode` is `Shl` and `FromCIValue` is 0, it means `X << 0`.
