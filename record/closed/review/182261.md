## Test Strategies

### Strategy 1: Test with profile data and dead cases

- **Target**: A switch instruction with a `umin` condition, some dead cases (e.g., `umin(x, 3)` with cases 0, 1, 2, 3, 4), and `!prof` branch weights metadata.
- **Rationale**: The fix changes `I = SIW->removeCase(I)` to `I = SIW.removeCase(I)`. `SIW` is a `SwitchInstProfUpdateWrapper`. `SIW->removeCase(I)` calls `SwitchInst::removeCase(I)`, which removes the case but does NOT update the profile data (branch weights). `SIW.removeCase(I)` calls `SwitchInstProfUpdateWrapper::removeCase(I)`, which removes the case AND updates the profile data. If there are dead cases and profile data is present, the old code would leave the profile data out of sync with the number of cases. The new code fixes this. We should test a switch with `umin` where some cases are dead (greater than the constant) and the switch has `!prof` metadata.
- **Expected Issue**: The branch weights metadata will be incorrect or malformed, potentially causing a crash or incorrect profile-guided optimization later.

### Strategy 2: Test with profile data, dead cases, and default case replacement

- **Target**: A switch instruction with a `umin` condition, dead cases, the constant case present (so default is replaced), and `!prof` branch weights metadata.
- **Rationale**: The optimization also replaces the default destination if the case matching the constant is found. It calls `SIW.replaceDefaultDest(Case)` and `SIW.removeCase(Case)`. We should ensure that when dead cases are removed *and* the default case is replaced, the profile data is updated correctly for all these operations.
- **Expected Issue**: The branch weights metadata will be incorrect or malformed, potentially causing a crash or incorrect profile-guided optimization later.

### Strategy 3: Test with profile data, no dead cases, but default case replacement

- **Target**: A switch instruction with a `umin` condition, no dead cases (e.g., `umin(x, 3)` with cases 0, 1, 2, 3), and `!prof` branch weights metadata.
- **Rationale**: Even if there are no dead cases, the default case might be replaced. The fix specifically addresses the loop removing dead cases, but it's important to verify the whole transformation maintains correct profile data.
- **Expected Issue**: The branch weights metadata will be incorrect or malformed, potentially causing a crash or incorrect profile-guided optimization later.

### Strategy 4: Test without profile data

- **Target**: A switch instruction with a `umin` condition and dead cases, but NO `!prof` branch weights metadata.
- **Rationale**: The fix changes how cases are removed to use the wrapper. The wrapper should correctly handle the case where there is no profile data. We should ensure this doesn't introduce a regression or crash when `!prof` is missing.
- **Expected Issue**: The optimization should not crash and should correctly handle the absence of profile data.

### Strategy 5: Test with profile data and multiple dead cases

- **Target**: A switch instruction with a `umin` condition, multiple dead cases (e.g., `umin(x, 3)` with cases 4, 5, 6), and `!prof` branch weights metadata.
- **Rationale**: The loop removes multiple dead cases. We should ensure that removing multiple cases in sequence correctly updates the profile data each time, as `SwitchInstProfUpdateWrapper::removeCase` modifies the weights array by swapping with the last element and popping.
- **Expected Issue**: The branch weights metadata will be incorrect or malformed, potentially causing a crash or incorrect profile-guided optimization later.
