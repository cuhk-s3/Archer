### Fix Summary
The patch fixes an issue in InstCombine where the transformation `fmul tan(X), cos(X) -> sin(X)` was being applied even when the `tan` and `cos` instructions lacked the `contract` fast-math flag. The transformation is a contraction, and it should only be applied if all involved operations (`tan`, `cos`, and `fmul`) have the `contract` flag explicitly set. The patch adds checks to ensure that both `tan` and `cos` have the `contract` flag before performing the transformation.

I have tested the following scenarios:
1. `fmul contract (tan contract X), (cos X)`: The transformation is correctly prevented because `cos` lacks the `contract` flag.
2. `fmul contract (tan X), (cos contract X)`: The transformation is correctly prevented because `tan` lacks the `contract` flag.
3. `fmul contract (tan X), (cos X)`: The transformation is correctly prevented because both `tan` and `cos` lack the `contract` flag.
4. `fmul (tan contract X), (cos contract X)`: The transformation is correctly prevented because `fmul` lacks the `contract` flag.
5. `fmul contract (cos X), (tan contract X)`: The transformation is correctly prevented because `cos` lacks the `contract` flag, even when the operands are commuted.

The fix is robust and correctly handles all these edge cases. No bugs were found.
