I have tested all the strategies on the provided test cases. The fix correctly prevents merging of global variables with non-dbg metadata in both `GlobalMerge` and `ConstantMerge` passes. It also correctly allows merging of global variables with ONLY `!dbg` metadata. No bugs were found in the fix.

Summary of the fix:
The patch introduces a new method `GlobalObject::hasMetadataOtherThanDebugLoc()` to check if a global object has any metadata other than `!dbg`. This method is then used in both `GlobalMerge` and `ConstantMerge` passes to prevent merging of global variables with non-dbg metadata. This is because merging globals with non-dbg metadata (like `!type`, `!absolute_symbol`, `!associated`, etc.) is not necessarily correct, as the semantics of such metadata might not be preserved or might be invalid when applied to the merged global. The `copyMetadata` function only knows how to correctly copy `!dbg` metadata. The fix ensures that only globals with no metadata or only `!dbg` metadata are merged, preventing potential miscompilations or loss of metadata semantics.
