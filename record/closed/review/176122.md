## Test Strategies

### Strategy 1: Test frexp mantissa with finite non-zero input

- **Target**: A function calling `frexp` on a finite non-zero value, extracting the mantissa, and demanding its FP class.
- **Rationale**: The patch adds `if (Known.isKnownAlways(fcInf | fcNan)) return II->getArgOperand(0);`. This is correct because for Inf and NaN, `frexp` returns the input as the mantissa. However, if the logic is flawed and somehow triggers for finite values, it would incorrectly return the input. We should test that finite values are NOT folded to the input.
- **Expected Issue**: The optimization might incorrectly fold the `frexp` call to its input argument, ignoring the fact that `frexp` normalizes the mantissa to the range [0.5, 1.0) (or [-1.0, -0.5)). If the input is a finite non-zero value, the mantissa is not equal to the input.

### Strategy 2: Test frexp mantissa with zero input

- **Target**: A function calling `frexp` on a value known to be zero, extracting the mantissa, and demanding its FP class.
- **Rationale**: The patch propagates `fcZero` from the input to the mantissa. If the input is known to be zero, the mantissa should be known to be zero. We should test this propagation.
- **Expected Issue**: The optimization might incorrectly assume the mantissa cannot be zero if the input is known not to be zero. However, `frexp` returns 0 if the input is 0. If the input can be 0, the mantissa can be 0. The patch correctly propagates `fcZero` from the input. We should test that if the input is known to be zero, the mantissa is known to be zero.

### Strategy 3: Test frexp mantissa with positive input

- **Target**: A function calling `frexp` on a value known to be positive, extracting the mantissa, and demanding its FP class.
- **Rationale**: The patch propagates `fcNegative` from the input to the mantissa. If the input is known to be positive, the mantissa should be known to be positive. We should test this propagation.
- **Expected Issue**: The optimization might incorrectly assume the mantissa cannot be negative if the input is known not to be negative. The patch correctly propagates `fcNegative` from the input. We should test that if the input is known to be positive, the mantissa is known to be positive.

### Strategy 4: Test frexp mantissa with negative input

- **Target**: A function calling `frexp` on a value known to be negative, extracting the mantissa, and demanding its FP class.
- **Rationale**: The patch propagates `fcPositive` from the input to the mantissa. If the input is known to be negative, the mantissa should be known to be negative. We should test this propagation.
- **Expected Issue**: The optimization might incorrectly assume the mantissa cannot be positive if the input is known not to be positive. The patch correctly propagates `fcPositive` from the input. We should test that if the input is known to be negative, the mantissa is known to be negative.

### Strategy 5: Test frexp mantissa is never subnormal

- **Target**: A function calling `frexp`, extracting the mantissa, and demanding its FP class, specifically checking if it can be subnormal.
- **Rationale**: The patch explicitly sets `Known.knownNot(fcSubnormal);`. This is because `frexp` normalizes the mantissa to the range [0.5, 1.0) (or [-1.0, -0.5)), so it can never be subnormal. We should test this property.
