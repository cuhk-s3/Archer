## Test Strategies

### Strategy 1: Division by Zero on Targets without Vector Registers

- **Target**: Compile a matrix multiplication on a target with no vector registers (e.g., `-mtriple=i386-unknown-linux-gnu -mattr=-sse`).
- **Rationale**: The `getNumOps` function divides by `double(TTI.getRegisterBitWidth(TargetTransformInfo::RGK_FixedWidthVector).getFixedValue())`. On targets without vector registers (e.g., x86 without SSE), `getRegisterBitWidth` returns 0. This results in division by 0.0, producing infinity, which causes undefined behavior when cast to `unsigned`.
- **Expected Issue**: Compiler crash due to floating-point division by zero or undefined behavior when casting infinity to unsigned.

### Strategy 2: Integer Overflow in Operation Counting

- **Target**: Matrix multiplication with very large dimensions (e.g., 4000x4000x4000) that would cause `NumFMAs` to exceed `UINT_MAX`.
- **Rationale**: `getNumNativeVectorOps` calculates `NumFMAs = C * getNumOps(EltType, R) * M`. For very large matrices (e.g., 4000x4000), this calculation can overflow a 32-bit `unsigned` integer. The wrapped-around value might be smaller than `TileLoopsThreshold`, causing the compiler to incorrectly bypass loop tiling.
- **Expected Issue**: The compiler will attempt to fully unroll a massive matrix multiplication, leading to Out-Of-Memory (OOM) or extreme compile times.

### Strategy 3: Large Matrices Not Divisible by TileSize

- **Target**: Matrix multiplication with large dimensions that are not multiples of the default `TileSize` (4), such as 1001x1001.
- **Rationale**: The condition for using loops is `(NumOps > TileLoopsThreshold) && R % TileSize == 0 && C % TileSize == 0`. If a matrix is extremely large but its dimensions are not multiples of `TileSize` (e.g., 1001x1001), the compiler will fall back to the fully unrolled code generation path.
- **Expected Issue**: The compiler will fall back to fully unrolling the matrix multiplication, causing OOM or extreme compile times.

### Strategy 4: Modulo by Zero with TileSize = 0

- **Target**: Compile any matrix multiplication with the command-line flag `-fuse-matrix-tile-size=0`.
- **Rationale**: The user can specify the tile size via the `-fuse-matrix-tile-size` command-line option. If the user sets this to 0, the condition `R % TileSize == 0` will result in an integer modulo by zero.
- **Expected Issue**: Compiler crash due to integer modulo by zero.
