## Test Strategies

### Strategy 1: Test with branch weights metadata

- **Target**: Add `!prof` metadata to the `select` instruction in `test_select_frexp_basic` or create a new test case with `!prof` metadata.
- **Rationale**: The fix specifically addresses the preservation of metadata (like branch weights) when folding `frexp` of `select`. If `ProfcheckDisableMetadataFixes` is false, the metadata from the original `select` should be preserved on the new `select`. We should test that branch weights are correctly preserved.
- **Expected Issue**: The select instruction will have incorrect branch weights metadata, potentially causing issues in downstream passes or incorrect profiling information.

### Strategy 2: Test with unpredictable metadata

- **Target**: Add `!unpredictable` metadata to the `select` instruction in a test case.
- **Rationale**: Similar to branch weights, `!unpredictable` metadata on the `select` instruction should also be preserved by `CreateSelectFMF` if `MDFrom` is provided. We should test that this metadata is preserved.
- **Expected Issue**: The select instruction will have unpredictable metadata, which might be lost if not properly preserved.

### Strategy 3: Test with ProfcheckDisableMetadataFixes enabled

- **Target**: Create a new RUN line in `select_frexp.ll` with `-profcheck-disable-metadata-fixes` and verify that `!prof` metadata is NOT preserved.
- **Rationale**: The fix explicitly checks `ProfcheckDisableMetadataFixes`. If this flag is true, the metadata should *not* be preserved. We should test this behavior using the `-profcheck-disable-metadata-fixes` flag in `opt`.
- **Expected Issue**: The select instruction will have incorrect metadata if `ProfcheckDisableMetadataFixes` is true, or the test will fail if the metadata is unexpectedly preserved when it shouldn't be.

### Strategy 4: Test with vector types and metadata

- **Target**: Add `!prof` metadata to the `select` instruction in `test_select_frexp_vec_splat`.
- **Rationale**: The fold applies to vector types as well. We should ensure that metadata preservation works correctly for vector `select` instructions.
- **Expected Issue**: The select instruction might have incorrect fast-math flags or metadata if the vector types are not handled correctly.
