## Test Strategies

### Strategy 1: Test VPlan Verification During Optimization

- **Target**: A test case that triggers `VPlanTransforms::optimize` with `-vplan-verify-each` enabled.
- **Rationale**: The patch replaces individual `runPass` calls inside `VPlanTransforms::optimize` with `RUN_VPLAN_PASS`. Previously, `runPass` was called directly, which verified the VPlan after each pass if `VerifyEachVPlan` was true. Now, `RUN_VPLAN_PASS` is used, which also verifies the VPlan. However, some passes inside `optimize` might leave the VPlan in a temporarily invalid state that is only fixed by subsequent passes. If `VerifyEachVPlan` is enabled, this could lead to verification failures.
- **Expected Issue**: The compiler will crash or assert because `verifyVPlanIsValid` is called on an invalid VPlan during the intermediate stages of `optimize`.
