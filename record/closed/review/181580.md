## Test Strategies

### Strategy 1: Basic ICmp Inversion

- **Target**: A sequence of `icmp eq` and `icmp ne` feeding into a `select` with `zext` operands.
- **Rationale**: Tests the core logic of the patch: identifying alternate `icmp eq`/`ne` compares, inverting the predicate, and swapping the `select` operands.
- **Expected Issue**: The optimization should trigger, unifying the compares and removing alternate shuffles. If it fails, the code will have extra shuffles. If it miscompiles, the swapped operands might produce incorrect results.

### Strategy 2: FCmp Inversion with Fast-Math Flags

- **Target**: A sequence of `fcmp oeq` and `fcmp une` with `nnan` flags feeding into a `select`.
- **Rationale**: The patch handles `FCmp` as well as `ICmp`. Floating-point compares have more complex inversion rules (ordered vs. unordered). We need to ensure flags like `nnan` are preserved or handled correctly during inversion.
- **Expected Issue**: The optimization should correctly invert `oeq` to `une` (or similar) and preserve the `nnan` flag. If it drops the flag or inverts incorrectly, it could lead to suboptimal or incorrect floating-point code.

### Strategy 3: Shared Select Operands

- **Target**: A `select` instruction where the true and false operands share values or are used in other vectorization trees.
- **Rationale**: The patch updates the `ValueToGatherNodes` map when swapping operands. If the true and false operands share values (e.g., `select c, A, A` or values used in other gathers), the update logic might be stressed.
- **Expected Issue**: The `UpdateGatherEntry` logic might fail to correctly update the mapping if values are shared, leading to a crash during vectorization or incorrect gather masks.

### Strategy 4: Poison/Undef in Gathers

- **Target**: A `select` instruction where some of the true/false operands are `poison` or `undef`.
- **Rationale**: Gather nodes often contain `poison` or `undef` for padding or unused lanes. Swapping operands with `poison`/`undef` needs to be handled carefully to avoid making the program more poisonous.
- **Expected Issue**: The swapping logic might mishandle `poison`/`undef`, either crashing during the `UpdateGatherEntry` phase or generating incorrect shuffle masks that propagate `poison` incorrectly.

### Strategy 5: ICmp with SameSign Flag

- **Target**: A sequence of `icmp samesign eq` and `icmp samesign ne` feeding into a `select`.
- **Rationale**: The patch explicitly preserves the `samesign` flag when creating the inverted predicate (`CmpPredicate InversedPred(CmpInst::getInversePredicate(Pred), Pred.hasSameSign());`). We should test this specific path.
- **Expected Issue**: The `samesign` flag should be preserved on the inverted compare. If it is dropped, the optimization might lose information. If it is incorrectly applied, it might cause a miscompile.
