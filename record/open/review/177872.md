## Test Strategies

### Strategy 1: Select without FMF, FCmp with FMF

- **Target**: A select instruction returning a floating-point value, with no fast-math flags, whose condition is an fcmp instruction with fast-math flags (e.g., `nnan`, `ninf`).
- **Rationale**: The fix intersects the FMF of the select and the fcmp. If the select has no FMF (or is not an FPMathOperator) and the fcmp has FMF, the intersection should result in no FMF (or only the FMF present in both). We need to ensure that the resulting fcmp and select do not inherit the fcmp's FMF if the select didn't have them.
- **Expected Issue**: The optimized select instruction will have fast-math flags that were not present in the original select instruction, potentially leading to incorrect optimizations later on (e.g., assuming no NaNs when NaNs are possible).

### Strategy 2: Select with FMF, FCmp without FMF

- **Target**: A select instruction returning a floating-point value, with fast-math flags, whose condition is an fcmp instruction without fast-math flags.
- **Rationale**: If the select has FMF but the fcmp does not, the intersection should result in no FMF. We need to ensure that the resulting fcmp and select do not inherit the select's FMF if the fcmp didn't have them.
- **Expected Issue**: The optimized select instruction will have fast-math flags that were not present in the original fcmp instruction, potentially leading to incorrect optimizations later on.

### Strategy 3: Select and FCmp with disjoint FMF

- **Target**: A select instruction returning a floating-point value, with some fast-math flags (e.g., `nnan`), whose condition is an fcmp instruction with different fast-math flags (e.g., `ninf`).
- **Rationale**: If the select has one set of FMF (e.g., `nnan`) and the fcmp has a disjoint set (e.g., `ninf`), the intersection should be empty. We need to ensure that the resulting fcmp and select have no FMF.
- **Expected Issue**: The optimized select instruction will have the union of the fast-math flags instead of the intersection, potentially leading to incorrect optimizations later on.

### Strategy 4: Select and FCmp with overlapping FMF

- **Target**: A select instruction returning a floating-point value, with some fast-math flags (e.g., `nnan ninf`), whose condition is an fcmp instruction with overlapping fast-math flags (e.g., `nnan nsz`).
- **Rationale**: If the select has one set of FMF (e.g., `nnan ninf`) and the fcmp has an overlapping set (e.g., `nnan nsz`), the intersection should be the common flags (`nnan`). We need to ensure that the resulting fcmp and select have only the common FMF.
- **Expected Issue**: The optimized select instruction will have fast-math flags that were not present in the original fcmp instruction, potentially leading to incorrect optimizations later on.

### Strategy 5: Non-FP Select, FCmp with FMF

- **Target**: A select instruction returning a non-floating-point value (e.g., an integer), whose condition is an fcmp instruction with fast-math flags.
- **Rationale**: If the select does not return a floating-point value (e.g., it returns an integer or a pointer), it cannot have FMF. The fix explicitly handles this by treating non-FP selects as having no FMF. We need to ensure that the resulting fcmp and select have no FMF.
- **Expected Issue**: The optimized select instruction will have fast-math flags, which is incorrect because the original select was not an FPMathOperator and therefore had no FMF.
