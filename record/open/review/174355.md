### Strategy 1: Samesign ULE with Negative Constant

- **Target**: Create a test with `icmp samesign ule i8 %x, -5` and check the computed constant range.
- **Rationale**: The patch intersects the unsigned range with the signed range when `samesign` is present. For `icmp samesign ule i8 %x, -5` (where -5 is 251 unsigned), the unsigned range is `[0, 252)`. The signed range for `sle %x, -5` is `[-128, -4)`. The intersection of `[0, 252)` and `[-128, -4)` might be empty or incorrect if the ranges wrap or are not handled correctly by `intersectWith`. Wait, `samesign` means both operands have the same sign. If the constant is negative, `%x` must be negative. So `%x` is in `[-128, -5]`. The unsigned range is `[0, 252)`. The signed range is `[-128, -4)`. The intersection should be `[-128, -4)`. Let's test if `intersectWith` handles this correctly.
- **Expected Issue**: The constant range computed for `%x` will be incorrect (e.g., empty or overly restrictive), leading to incorrect optimization (e.g., folding a valid branch to unreachable or replacing `%x` with a constant).
