## Test Strategies

### Strategy 1: foldFDivSqrtDivisor with negative zero

- **Target**: `foldFDivSqrtDivisor` in `InstCombineMulDivRem.cpp`
- **Rationale**: The transformation `X / sqrt(Y / Z) --> X * sqrt(Z / Y)` is performed when `reassoc` and `arcp` are present. However, if `Y / Z` is `-0.0`, `sqrt(-0.0)` is `-0.0`. `X / -0.0` is `-Inf` (for `X > 0`). In the transformed expression, `Z / Y` is `-Inf`, `sqrt(-Inf)` is `NaN`, and `X * NaN` is `NaN`. This changes the result from `-Inf` to `NaN`. The transformation should require `nsz` or `nnan` to be safe.
- **Expected Issue**: The result might be NaN instead of -Inf, or vice versa, violating the semantics of the original program.
