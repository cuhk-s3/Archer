## Test Strategies

### Strategy 1: Strong SIV with Src wrapping (no nsw)

- **Target**: A loop with two array accesses where the source access index is an AddRec without `nsw` (e.g., large step that overflows) and the destination access index is an AddRec with `nsw`.
- **Rationale**: If the source AddRec lacks the `nsw` flag, it can wrap around the address space. The linear equation used by Strong SIV assumes no wrapping. If it wraps, a dependence might exist that the test misses.
- **Expected Issue**: DependenceAnalysis incorrectly disproves dependence or computes an incorrect distance because it ignores the signed wrap of the source AddRec.

### Strategy 2: Strong SIV with Dst wrapping (no nsw)

- **Target**: A loop with two array accesses where the destination access index is an AddRec without `nsw` and the source access index is an AddRec with `nsw`.
- **Rationale**: Similar to the source wrapping, if the destination AddRec lacks the `nsw` flag, it can wrap around, leading to missed dependencies.
- **Expected Issue**: DependenceAnalysis incorrectly disproves dependence or computes an incorrect distance because it ignores the signed wrap of the destination AddRec.

### Strategy 3: Strong SIV with both Src and Dst wrapping (no nsw)

- **Target**: A loop with two array accesses where both source and destination access indices are AddRecs without `nsw`.
- **Rationale**: If both AddRecs lack the `nsw` flag, they can both wrap. The Strong SIV test must bail out to avoid incorrect dependence analysis.
- **Expected Issue**: DependenceAnalysis incorrectly disproves dependence because it ignores the signed wrap of both AddRecs.

### Strategy 4: Strong SIV with nuw but no nsw

- **Target**: A loop with two array accesses where the AddRecs have the `nuw` flag but lack the `nsw` flag.
- **Rationale**: The patch specifically checks for `hasNoSignedWrap()`. If an AddRec has `nuw` (no unsigned wrap) but not `nsw`, the Strong SIV test should still bail out because its internal math (e.g., `sdivrem`, `minusSCEVNoSignedOverflow`) relies on signed arithmetic properties.
- **Expected Issue**: DependenceAnalysis might incorrectly analyze the dependence if it relies on `nuw` instead of `nsw`, as the distance computation uses signed arithmetic.

### Strategy 5: Strong SIV with nsw on both (Baseline)

- **Target**: A loop with two array accesses where both source and destination access indices are AddRecs with the `nsw` flag.
- **Rationale**: To ensure the fix doesn't overly restrict the analysis, we should test the case where both AddRecs have the `nsw` flag. The Strong SIV test should proceed and correctly analyze the dependence.
- **Expected Issue**: DependenceAnalysis should successfully analyze the dependence and compute the correct distance/direction.
