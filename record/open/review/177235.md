## Test Strategies

### Strategy 1: NonNull Attribute on Call Site

- **Target**: A call site where an argument is marked `nonnull` and the callee checks if the argument is null.
- **Rationale**: The fix checks if the argument has the `nonnull` attribute at the call site. If it does, and the condition is an equality check against null, the branch can be folded. We should test a case where the call site has the `nonnull` attribute on the argument.
- **Expected Issue**: The dead block is not pruned during inlining, leading to larger code size and potentially missed optimizations.

### Strategy 2: NonNull Attribute on Callee Parameter

- **Target**: A call site where the callee parameter is marked `nonnull` and the callee checks if the argument is null.
- **Rationale**: The fix uses `CB.paramHasAttr(A->getArgNo(), Attribute::NonNull)`. `paramHasAttr` checks both the call site and the called function's parameters. We should test a case where the callee parameter has the `nonnull` attribute, but the call site does not explicitly have it.
- **Expected Issue**: The dead block is not pruned during inlining, leading to larger code size and potentially missed optimizations.

### Strategy 3: No NonNull Attribute

- **Target**: A call site where neither the argument nor the callee parameter is marked `nonnull`, and the callee checks if the argument is null.
- **Rationale**: If neither the call site nor the callee parameter has the `nonnull` attribute, the branch should not be folded. We should test this to ensure the optimization does not trigger incorrectly.
- **Expected Issue**: The dead block is pruned incorrectly, leading to incorrect execution if the argument can actually be null.

### Strategy 4: NonNull Attribute with ICMP_NE

- **Target**: A call site where an argument is marked `nonnull` and the callee checks if the argument is not null (`ICMP_NE`).
- **Rationale**: The fix checks `CI->isEquality()`. It handles both `ICMP_EQ` and `ICMP_NE`. We should test a case where the condition is `ICMP_NE` against null.
- **Expected Issue**: The dead block is not pruned during inlining, leading to larger code size and potentially missed optimizations.

### Strategy 5: NonNull Attribute with Non-Null Constant

- **Target**: A call site where an argument is marked `nonnull` and the callee checks if the argument is equal to a non-null constant pointer.
- **Rationale**: The fix checks `isa<ConstantPointerNull>(CI->getOperand(1))`. If the comparison is against a non-null constant, the branch should not be folded based on the `nonnull` attribute alone. We should test this to ensure the optimization does not trigger incorrectly.
- **Expected Issue**: The dead block is pruned incorrectly, leading to incorrect execution.
