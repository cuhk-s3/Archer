## Test Strategies

### Strategy 1: Same-direction shift under Unsigned semantics must not fold

- **Target**: Mutate/extend the new icmp-shl-lshr regression shape to multiple unsigned predicates (`uge`,`ugt`,`ule`,`ult`), scalar and vector types, and varied shift amounts including non-equal and equal amounts.
- **Rationale**: The reland fixes exactly this class by requiring `Lossy` for same-direction shift composition. Mutating regression to nearby predicates/types/constants can reveal missed paths where same-direction `lshr` is still accepted.
- **Expected Issue**: Incorrect fold of `icmp uge (shl nuw X, C), (lshr Y, D)` into `icmp uge X, (lshr Y, C+D)` (or similar), changing truth values.

### Strategy 2: Signed semantics boundary for opposite-direction equal shifts

- **Target**: Create/modify tests for `icmp s* (shl nsw X, C), Op1` with `Op1` as `lshr (shl Y, C), C` under variants: with/without `nsw`, equal vs non-equal shift amounts, and signed predicates only.
- **Rationale**: `canEvaluateShiftedShift` now has a special signed path (`!IsOuterShl && Signed`) requiring inner `shl nsw` and equal amounts. This is subtle and easy to mishandle in other shapes.
- **Expected Issue**: Wrong transform in signed compares where `lshr(shl nsw Y, C), C` is treated as lossy-and-mask instead of exact `Y`, or vice versa.

### Strategy 3: Add-wrap-flag gating must be strict

- **Target**: Mutate existing icmp-shl tests to use RHS `add (shl Y, C), K` with combinations of add flags: none / wrong flag / correct flag / both flags; include constants K divisible and non-divisible by `2^C`.
- **Rationale**: New Add support in `canEvaluateShifted` allows right-shift only when wrap flag matches semantics (`nuw` for unsigned, `nsw` for signed). Any leakage here miscompiles around wrap boundaries.
- **Expected Issue**: Fold occurs despite missing `nuw/nsw` on inner `add`, yielding non-equivalent compare after right-shift peel.

### Strategy 4: Constant divisibility and poison lane robustness

- **Target**: Add vector tests (`<N x iM>`) with constant RHS addends and/or direct constants: divisible in some lanes, indivisible in others, and poison elements (`<..., poison, ...>`), verifying no unsafe fold.
- **Rationale**: For non-lossy right-shift, constants are only legal when divisible by `2^NumBits` (`countr_zero >= NumBits`). Vector poison/splat handling can accidentally bypass this check.
- **Expected Issue**: Incorrect fold with constants/poison splats where low-bit divisibility precondition is not enforced lane-wise, especially vectors.

### Strategy 5: Overshift and non-power-of-two width stress

- **Target**: Mutate shift/icmp tests to i33/i65 and vectors thereof, with boundary shift amounts (`0`, `bw-1`, `bw`) embedded in RHS trees (`add`, `lshr`, `shl`, `phi/select`) to ensure no transform or crash when illegal.
- **Rationale**: The fold checks `ShAmt < bitwidth` on Op0, but recursive `getShiftedValue/foldShiftedShift` manipulates inner shifts and masks; unusual widths (e.g., i33) and boundary shift constants can expose missing guards.
- **Expected Issue**: Potential compile-time crash or invalid IR creation when shift amount equals/overflows bitwidth in nested shift/add trees during ICmp fold path.
