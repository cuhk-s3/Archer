## Test Strategies

### Strategy 1: Hyperbolic parity matrix

- **Target**: Add IR tests for cosh/coshf/coshl, sinh/sinhf/sinhl, tanh/tanhf/tanhl with fneg operands and check exact transformed shape.
- **Rationale**: The change is switch-case dispatch only; easy to miss one variant or assign wrong even/odd bucket.
- **Expected Issue**: Some newly added LibFunc variants may fail to fold or fold with wrong sign/parity.

### Strategy 2: StrictFP blocking

- **Target**: Create strictfp calls to sinh/cosh/tanh of negated args and check no canonicalization happens.
- **Rationale**: The optimization must not run when CI is strictfp; verify new hyperbolic entries respect the existing guard.
- **Expected Issue**: Rewrite could incorrectly occur in strict FP mode.

### Strategy 3: NoBuiltin blocking

- **Target**: Add nobuiltin sinh/cosh/tanh calls with fneg args and check original calls remain unchanged.
- **Rationale**: LibCallSimplifier exits early on nobuiltin; ensure new paths are not reachable.
- **Expected Issue**: Rewrite could incorrectly occur on nobuiltin calls.

### Strategy 4: NaN sign observability stress

- **Target**: Use NaN-producing input to sinh/tanh with sign-observing follow-up ops (copysign/bitcast paths) and inspect whether rewrite is still performed.
- **Rationale**: f(-x)->-f(x) may differ for NaN sign/payload behaviors in some libm implementations and observable downstream sign usage.
- **Expected Issue**: Potential semantic mismatch for odd-function rewrite when NaN sign observability matters.

### Strategy 5: Asinh/atanh gap probe

- **Target**: Add asinh/atanh (and f/l variants) with fneg arg to document current non-folding behavior and catch future regressions/intent changes.
- **Rationale**: Patch says hyperbolic lib functions but only sinh/cosh/tanh families were added; asinh/atanh are odd too.
- **Expected Issue**: Coverage gap: other odd hyperbolic functions remain unsimplified.

declare double @sinh(double)
declare double @cosh(double)
declare double @tanh(double)

define double @odd_even_matrix(double %x) {
  %nx = fneg double %x
  %a = call double @sinh(double %nx)
  %b = call double @cosh(double %nx)
  %c = call double @tanh(double %nx)
  %s = fadd double %a, %b

### Analysis

No bug found after exhaustive Phase-2 verification across all managed tests and all proposed strategies.

Bug Triggering Analysis:
- I exercised all provided hyperbolic neg-arg patterns (double/float/fp128, fneg and fsub -0.0 forms, plus extra-use OneUse blocker form) and observed expected InstCombine rewrites:
  - odd: sinh/tanh/tanhl/sinhf/sinhl/tanhf => call on positive arg then fneg result
  - even: cosh/coshf/coshl => call on positive arg
  - even+fabs: cosh(fabs(x)) => cosh(x)
  - OneUse guard: when negated operand has extra use, rewrite does not fire.
- Guard behavior was checked repeatedly:
  - callsite nobuiltin attribute prevented simplify-libcalls rewrites as expected.
  - strictfp function attribute did NOT block these rewrites in observed transforms (rewrites still occurred), but differential testing showed no behavioral divergence for tested concrete inputs; this appears to reflect current pass behavior rather than a proven miscompile from this patch.
- NaN-sign observability stress (with copysign consumers) produced no differential output differences for tested constants.
- Asinh/atanh gap probes confirmed no new symmetric fold for these functions (unchanged behavior), i.e., coverage gap/intent mismatch but not a regression introduced by this patch.

Fix Weakness Analysis:
- The patch itself is a switch-dispatch extension in optimizeSymmetric; tests confirm newly added entries are wired correctly by parity and type-family coverage.
- Potential weakness remains around strictfp expectations (transforms still happen under strictfp-attributed functions in these checks), but this appears broader than this patch and was not shown as a concrete wrong-code bug here.
- Therefore, based on current verification and differential execution, the submitted fix appears semantically stable for targeted scenarios; no actionable bug repro was found.
