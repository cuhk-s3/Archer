## Test Strategies

### Strategy 1: Non-byte-sized byte load poison semantics

- **Target**: Mutate constant-load tests to load b1/b7 from constant globals containing 0xFF/0x7F bytes via bitcasted pointers.
- **Rationale**: New byte-enabled reinterpret fold maps byte loads through integers. For bN where N is not a multiple of 8, folding may truncate non-zero discarded bits and return a concrete constant, but LangRef requires poison in that case.
- **Expected Issue**: Incorrect constant result instead of poison when truncation from byte-sized memory should poison.

### Strategy 2: Aggregate ConstantByte recursion coverage

- **Target**: Mutate load-fold tests to use [2 x b8] or { b8, b8 } initializers and load b16/i16 across elements.
- **Rationale**: ReadDataFromGlobal has explicit scalar handling for ConstantInt/ConstantFP; byte constants inside aggregates rely on recursive/aggregate paths and may fail unexpectedly.
- **Expected Issue**: Missed fold for byte aggregates even though load is from constant global.

### Strategy 3: Big-endian reinterpret ordering for byte loads

- **Target**: Clone an existing load-fold test with big-endian datalayout and verify exact constants for cross-byte loads.
- **Rationale**: Byte loads now reach endian-sensitive reinterpret assembly more often. Existing tests are mostly little-endian.
- **Expected Issue**: Wrong folded constant on big-endian targets.

### Strategy 4: Uniform all-ones -> byte vector fold

- **Target**: Mutate evaluate-load-uniform style tests to load <N x b8> from all-ones uniform constants through offset GEPs and expect constant result.
- **Rationale**: Patch explicitly adds byte/byte-vector in ConstantFoldLoadFromUniformValue; this path should now fold in InstSimplify/GlobalOpt.
- **Expected Issue**: Missed all-ones fold for byte vectors from uniform globals.

### Strategy 5: Uniform all-ones with non-byte-sized byte types

- **Target**: Add tests using b1/b3 all-ones uniform initializers and byte destinations; verify no invalid fold (or poison where required).
- **Rationale**: Uniform folding assumes source type has equal alloc/store size; byte types with unusual widths stress this precondition and may expose unsound all-ones materialization.
- **Expected Issue**: Over-fold on non-byte-sized uniform byte types where conservative behavior is required.
