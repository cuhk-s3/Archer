## Test Strategies

### Strategy 1: Vector poison/undef element constants

- **Target**: Mutate scalar tests to <2 x i8>/<4 x i8> vector forms using constants such as <i8 -128, i8 poison>, <i8 127, i8 poison>, and <i8 0, i8 poison>; optionally freeze the select result to make poison differences observable.
- **Rationale**: m_MinSignedValue, m_MaxSignedValue, and m_ZeroInt are cst_pred_ty matchers that ignore poison elements in fixed vector constants by default. The new fold combines three such matches, so a vector pattern with poison in the compare RHS, true value, or zero operand can be accepted even though per-lane semantics of the original select may be poison rather than a concrete saturated negation.
- **Expected Issue**: The fold may treat poison elements in vector constants as matching concrete MIN/MAX/0 values and replace a lane whose original select result was poison/undef-dependent with a concrete saturating result, changing poison/refinement behavior observable via freeze or poison-sensitive uses.

### Strategy 2: Guarded flagged negation

- **Target**: Mutate the false arm to sub nsw, sub nuw, and sub nuw nsw variants; add extra uses and freeze of the select result to reveal poison differences.
- **Rationale**: The matcher m_Sub(m_ZeroInt(), m_Specific(CmpLHS)) does not reject nsw/nuw. A false arm like sub nsw 0, %x is poison at %x == MIN_INT, but selected only when %x != MIN_INT, so nsw is safe for signed negation. However sub nuw 0, %x is poison for almost every nonzero %x, including many selected false-arm cases. Folding it to unflagged ssub_sat 0, %x can replace poison with a concrete value.
- **Expected Issue**: The fold may be applied to an original false arm that is poison-generating under flags on inputs other than MIN_INT, producing a non-poison ssub_sat result and changing behavior where the original select would return poison.

### Strategy 3: Predicate and operand order variants

- **Target**: Create variants with icmp ne, with min constant on the left side of the icmp, and with true/false arms swapped; include negative tests where saturation constant or selected arm is off by one.
- **Rationale**: The code only normalizes ICMP_NE by swapping select arms. It checks CmpRHS for MIN_INT and FalseVal for 0 - CmpLHS. Equivalent IR can use icmp eq MIN_INT, %A, icmp ne with arms arranged as %A != MIN ? 0-A : MAX, or non-equivalent near misses with swapped selected arms. Tests are needed to ensure only semantically equivalent forms fold.
- **Expected Issue**: InstCombine may miss valid variants or incorrectly fold invalid ones depending on predicate and operand order; wrong arm swapping could produce ssub_sat when the selected MAX/negation arms are inverted.

### Strategy 4: Unusual integer widths including i1 and non-byte widths

- **Target**: Mutate tests to i1, i3, i4, i9, and i33 versions of x == min ? max : 0 - x; check correct llvm.ssub.sat formation or further simplification with no verifier errors.
- **Rationale**: The transformation relies on APInt signed min/max and creates llvm.ssub.sat with ConstantInt::getNullValue(CmpLHS->getType()). Non-standard widths often expose hidden assumptions in InstCombine folds. i1 is degenerate: MIN_INT is -1/true and MAX_INT is 0/false, making the expression constant-like but still representable as ssub_sat.
- **Expected Issue**: The fold may be wrong or crash for unusual integer widths if signed min/max/null constants or overloaded intrinsic declarations are constructed incorrectly, especially for i1 where signed min is true and signed max is false.

### Strategy 5: Specific operand mismatch and vector shape negatives

- **Target**: Create negative tests like %cmp = icmp eq %x, min; %neg = sub 0, %y; select %cmp, max, %neg, plus vector variants using shufflevector %x or different but equal-looking values.
- **Rationale**: Correctness requires that the value compared against MIN_INT is exactly the value negated in the false arm, with the same scalar/vector type. The matcher uses m_Specific(CmpLHS), so tests should ensure it does not fold when negation uses a different SSA value, a shuffled vector, or a value merely known equal to CmpLHS.
- **Expected Issue**: The fold may incorrectly match when the compared value and negated value differ, or miss required type/shape restrictions for vectors, leading to use of the wrong operand in the created intrinsic.
