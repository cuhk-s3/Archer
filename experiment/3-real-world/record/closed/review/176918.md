## Test Strategies

### Strategy 1: Out-of-bounds extractelement index in cost modeling

- **Target**: An `extractelement` instruction with a constant index that is greater than or equal to the number of elements in the vector operand.
- **Rationale**: The fix prevents an out-of-bounds index from being returned by `getExtractIndex`. If an out-of-bounds index is returned, it is used to set a bit in an APInt (`DemandedElts.setBit(Idx)`), which causes an assertion failure if the index is greater than or equal to the bitwidth of the APInt (which is the number of elements in the vector). By providing an out-of-bounds index, we can trigger this assertion failure in the unpatched code.
- **Expected Issue**: The compiler will crash due to an assertion failure in APInt when trying to set a bit at an out-of-bounds index.
