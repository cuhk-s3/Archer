The fix adds a new statistic `LargestFunctionBBCount` to the `InstCount` pass. This statistic tracks the maximum number of basic blocks in a single function across the module. The fix correctly updates the maximum value using `F.size()` in the `visitFunction` method.

I have tested the fix with various scenarios:
1. Multiple functions with different numbers of basic blocks.

3. Empty functions (only entry block).
4. Functions with a large number of basic blocks.
5. External functions (declarations).

In all cases, the `LargestFunctionBBCount` statistic correctly reflects the maximum number of basic blocks in a single non-external function. The fix is correct and does not introduce any bugs. The test cases I generated can be used to verify the correctness of the fix.
