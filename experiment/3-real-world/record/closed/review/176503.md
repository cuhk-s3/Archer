## Test Strategies

### Strategy 1: PHI node in second loop preheader used by non-PHI in exit block

- **Target**: Create two adjacent loops. In the preheader of the second loop, add a PHI node with a single incoming value from the first loop's exit block. Use this PHI node in a non-PHI instruction in the second loop's exit block.
- **Rationale**: When a PHI node in the second loop's preheader is only used in the exit block of the second loop, it will be sunk to the exit block. Before the fix, it was moved as a PHI node to the exit block, which makes it invalid since its incoming block is from the first loop, not the second loop's exiting blocks.
- **Expected Issue**: The compiler will crash with an assertion failure like 'Instruction does not dominate all uses' or a verifier error because the PHI node is moved to the exit block but its incoming block does not match the predecessors of the exit block.

### Strategy 2: PHI node in second loop preheader used by PHI in exit block

- **Target**: Create two adjacent loops. In the preheader of the second loop, add a PHI node. In the exit block of the second loop, add another PHI node that uses the first PHI node as an incoming value.
- **Rationale**: If the sunk PHI node is used by another PHI node in the exit block, moving it to the exit block might place it among other PHI nodes. The verifier will complain about the incoming block of the sunk PHI node not being a predecessor of the exit block. This directly mimics the crash in the reported issue.
- **Expected Issue**: The compiler will crash with a verifier error because the PHI node's incoming block is not a predecessor of the exit block, or an assertion failure during dominance check.

### Strategy 3: Multiple PHI nodes in second loop preheader

- **Target**: Add multiple PHI nodes in the preheader of the second loop, all used in the exit block of the second loop.
- **Rationale**: Testing multiple PHI nodes ensures that the fix correctly iterates and replaces all PHI nodes in the preheader without iterator invalidation issues or missing any PHI nodes.
- **Expected Issue**: The compiler will crash with a verifier error or dominance assertion failure for all the sunk PHI nodes.

### Strategy 4: PHI node in second loop preheader used further down the CFG

- **Target**: Create two adjacent loops. Add a PHI node in the second loop's preheader. Use this PHI node in a block that comes after the second loop's exit block.
- **Rationale**: If the PHI node is used in a block that is strictly dominated by the exit block (not the exit block itself), it will still be sunk to the exit block. The same invalid PHI node issue will occur.
- **Expected Issue**: The compiler will crash with a verifier error because the PHI node is moved to the exit block but its incoming block is invalid.
