## Test Strategies

### Strategy 1: Test nooutline attribute on a function with a cold region

- **Target**: Create a new test case or mutate an existing one (e.g., `llvm/test/Transforms/HotColdSplit/split-cold-2.ll`) by adding the `"nooutline"` attribute to the function and verifying that no outlining occurs.
- **Rationale**: The patch adds a check for the `nooutline` attribute in `HotColdSplitting::shouldOutlineFrom`. We need to verify that a function with this attribute is indeed skipped by the pass, even if it contains a clearly cold region that would otherwise be split.
- **Expected Issue**: The function with the `nooutline` attribute is still split by HotColdSplitting, violating the attribute's semantics.

### Strategy 2: Test function without nooutline attribute

- **Target**: Use an existing test case (e.g., `llvm/test/Transforms/HotColdSplit/split-cold-2.ll`) without the `"nooutline"` attribute and verify that outlining still occurs as expected.
- **Rationale**: We need to ensure that the addition of the `nooutline` check does not inadvertently affect functions that do not have the attribute. They should still be split if they contain cold regions.
- **Expected Issue**: The function without the `nooutline` attribute is not split, or the pass crashes.

### Strategy 3: Test NumFunctionsSkippedByNoOutlineAttr statistic

- **Target**: Run the pass with `-stats` on a test case containing a function with the `"nooutline"` attribute and verify that the statistic is incremented.
- **Rationale**: The patch adds a new statistic `NumFunctionsSkippedByNoOutlineAttr`. We should verify that this statistic is correctly incremented when a function is skipped due to the `nooutline` attribute.
- **Expected Issue**: The statistic `NumFunctionsSkippedByNoOutlineAttr` is not incremented correctly.

### Strategy 4: Test nooutline attribute combined with other attributes

- **Target**: Mutate a test case by adding both `"nooutline"` and another attribute like `"cold"` or `"optsize"` to the function, and verify that it is not split.
- **Rationale**: Functions can have multiple attributes. We need to ensure that the `nooutline` attribute takes precedence and prevents splitting even if other attributes might suggest otherwise (e.g., a function marked `cold` but also `nooutline`).
- **Expected Issue**: The `nooutline` attribute is ignored when combined with other attributes like `cold` or `optsize`.

### Strategy 5: Test multiple functions with and without nooutline

- **Target**: Create a test case with two functions, one with `"nooutline"` and one without, both containing cold regions. Verify that only the function without the attribute is split.
- **Rationale**: The pass operates on a module level. We need to ensure that the `nooutline` attribute is respected on a per-function basis, and that its presence on one function does not affect the splitting of other functions in the same module.
- **Expected Issue**: The `nooutline` attribute is ignored on a function that is part of a larger module with multiple functions, some with and some without the attribute.
