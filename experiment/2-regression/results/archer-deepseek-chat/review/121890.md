## Test Strategies

### Strategy 1: Multi-index wrap edge case

- **Target**: Test GEPs with multiple indices where intermediate offset calculations might wrap despite final offset being within bounds
- **Rationale**: EmitGEPOffset computes total offset, but nowrap flags apply to each addition step. The patch assumes total offset comparison is valid if GEP has nowrap flags, but intermediate wrap could violate semantics.
- **Expected Issue**: Incorrect folding when intermediate wrap occurs despite final offset being representable

### Strategy 2: Vector lane poison propagation

- **Target**: Vector GEPs with nowrap flags where some lanes would wrap
- **Rationale**: Vector nowrap flags apply element-wise; poison in one lane shouldn't affect others. The patch may incorrectly propagate poison or miss valid folds for non-poison lanes.
- **Expected Issue**: Incorrect poison propagation or missed folding opportunities for vector GEP comparisons

### Strategy 3: Mixed flag combinations

- **Target**: GEPs with different nowrap flag combinations (LHS nuw, RHS nusw, etc.)
- **Rationale**: Intersection logic (NW = GEPLHS->getNoWrapFlags() & GEPRHS->getNoWrapFlags()) may be too conservative or optimistic for certain mixed flag cases.
- **Expected Issue**: Missing valid folds or incorrect folds when GEPs have different nowrap flag sets

### Strategy 4: Type scaling overflow

- **Target**: Large indices with small element types causing offset multiplication overflow
- **Rationale**: nuw/nusw apply to scaled offset after type size multiplication, not raw index. The patch may incorrectly assume index comparison preserves wrap semantics.
- **Expected Issue**: Incorrect folding when scaled offset wraps despite index not wrapping

### Strategy 5: Address space interaction

- **Target**: GEPs with address space casts and nowrap flags
- **Rationale**: Different address spaces may have different pointer sizes/wrap semantics. The patch may incorrectly fold comparisons across address space boundaries.
- **Expected Issue**: Incorrect folding when nowrap semantics differ across address spaces
