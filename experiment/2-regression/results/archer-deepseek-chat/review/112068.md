## Test Strategies

### Strategy 1: Multi-use ctlz/cttz with mixed operations

- **Target**: Test cases where ctlz/cttz has multiple uses including shifts and other operations
- **Rationale**: The optimization assumes single-use, but if ctlz/cttz has multiple uses and we change zero_is_poison to true, we might poison other uses that rely on defined behavior for zero input
- **Expected Issue**: Poison propagation to non-shift uses causing miscompilation

### Strategy 2: Indirect shift amount through arithmetic

- **Target**: Test cases where ctlz/cttz result is modified (add/sub/zext/trunc) before being used as shift amount
- **Rationale**: The pattern matching only checks direct use as shift amount. If the value is transformed, the shift-by-bitwidth UB guarantee might not hold
- **Expected Issue**: Transformed shift amount might not trigger UB for zero input, making poison propagation unsafe

### Strategy 3: Vector lanes with mixed zero/non-zero inputs

- **Target**: Vector test cases where some lanes have zero input and others don't
- **Rationale**: For vectors, changing zero_is_poison to true might poison entire vector result even if only some lanes have zero input, affecting non-zero lanes
- **Expected Issue**: Poison propagation from zero lanes contaminating non-zero lanes in vector operations

### Strategy 4: Shift with poison-sensitive flags or metadata

- **Target**: Test cases where shift has nuw/nsw flags or range metadata
- **Rationale**: Changing ctlz/cttz to return poison might cause shift to generate poison from flags/metadata that were previously safe
- **Expected Issue**: Poison generation from shift flags/metadata when shift amount becomes poison

### Strategy 5: ctlz/cttz used in guarded contexts

- **Target**: Test cases where ctlz/cttz usage is guarded by conditions on the input
- **Rationale**: If the code has guards that prevent zero input, changing zero_is_poison might enable removal of those guards, but pattern doesn't check for guards
- **Expected Issue**: Removal of safety guards leading to poison propagation in previously safe code
