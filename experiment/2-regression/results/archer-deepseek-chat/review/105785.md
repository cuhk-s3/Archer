## Test Strategies

### Strategy 1: Wide Integer Type Overflow

- **Target**: Modify existing ucmp/scmp tests to use i128/i256 types with large constants
- **Rationale**: The constraint system uses 64-bit internal representation. Wide types with large constants can cause coefficient overflow during decomposition, leading to incorrect constraint modeling and wrong simplification of ucmp/scmp intrinsics.
- **Expected Issue**: Constraint system incorrectly proves relationships due to truncated coefficients, causing ucmp/scmp to be replaced with wrong constant (1, 0, or -1).

### Strategy 2: Poison Propagation from Overflow Operations

- **Target**: Add poison-generating operations (add nuw, mul nsw) as operands to ucmp/scmp
- **Rationale**: The fix doesn't check if operands are guaranteed not to be poison. If operands come from overflow operations that generated poison, the constraint system may incorrectly assume well-defined values and simplify the intrinsic.
- **Expected Issue**: Pass incorrectly simplifies ucmp/scmp when operands are poison, violating poison semantics.

### Strategy 3: Signed/Unsigned Predicate Confusion

- **Target**: Create tests where signed and unsigned relationships conflict near boundary values
- **Rationale**: The fix uses getGTPredicate()/getLTPredicate() which return different predicates for ucmp vs scmp. Edge cases near signed/unsigned boundaries (e.g., -1 vs UINT_MAX) could cause incorrect simplification if constraint system mixes signed/unsigned reasoning.
- **Expected Issue**: Pass incorrectly simplifies ucmp using signed constraints or scmp using unsigned constraints near value boundaries.

### Strategy 4: Type Size Mismatch in Constant Creation

- **Target**: Test ucmp/scmp with unusual return types (i1, i2, i4, i16)
- **Rationale**: ConstantInt::getSigned(I->getType(), -1) may not work correctly for small integer types where -1 representation differs. Also, the fix assumes return type can hold values 1, 0, -1.
- **Expected Issue**: Incorrect constant creation for small integer types or type mismatch errors.

### Strategy 5: Overflow in Constraint Coefficient Calculation

- **Target**: Use large constants in arithmetic expressions feeding ucmp/scmp operands
- **Rationale**: During decomposition, multiplyWithOverflow and addWithOverflow are used. Large constants can overflow 64-bit coefficients, corrupting the constraint model while appearing to work.
- **Expected Issue**: Silent overflow in coefficient calculations leads to incorrect constraint relationships and wrong ucmp/scmp simplification.
