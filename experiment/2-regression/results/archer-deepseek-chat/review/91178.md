### Strategy 2: Cross-Lane Vector Safety

- **Target**: Test scalar operations with vector operands that have lane-dependent semantics
- **Rationale**: The fix allows scalar operations on vector operands but some scalar ops may not be cross-lane safe
- **Expected Issue**: Incorrect simplification due to assuming cross-lane safety where it doesn't hold

### Strategy 3: Deep Poison Propagation

- **Target**: Test cases where impliesPoison analysis depth is insufficient for recursive replacements
- **Rationale**: impliesPoison has depth limit of 2, but recursive replacements could create deeper dependencies
- **Expected Issue**: Incorrect absorber simplification due to incomplete poison analysis

### Strategy 4: Constant Poison Creation

- **Target**: Test constant folding after replacement that creates poison from non-poison inputs
- **Rationale**: Constant folding may create poison (e.g., division by zero) that wasn't present before replacement
- **Expected Issue**: Introduction of poison values through constant folding after replacement

### Strategy 5: Type Compatibility in Recursion

- **Target**: Test recursive replacements that lead to type mismatches in operations
- **Rationale**: Recursive replacement could change operand types in ways that make operations invalid
- **Expected Issue**: Type errors or incorrect simplifications due to type incompatibilities

### Strategy 6: Absorber Case Over-Application

- **Target**: Test cases where either operand becomes absorber but simplification isn't valid
- **Rationale**: New check (NewOps[0] == Absorber || NewOps[1] == Absorber) may be too permissive without proper poison analysis
- **Expected Issue**: Incorrect simplification to absorber when poison could leak
