## Test Strategies

### Strategy 1: Vector with poison elements

- **Target**: Test vector constants containing poison elements in getFlippedStrictnessPredicateAndConstant
- **Rationale**: The function replaces undefs with safe constants but may not handle poison correctly. Poison elements could propagate incorrectly when flipping predicates.
- **Expected Issue**: Incorrect transformation due to mishandled poison elements leading to miscompilation

### Strategy 2: Constant at type boundaries

- **Target**: Test constants at min/max values (0, UINT_MAX, INT_MIN, INT_MAX) with strict predicates
- **Rationale**: getFlippedStrictnessPredicateAndConstant checks for overflow but edge cases may be missed, especially with vector constants or unusual types.
- **Expected Issue**: Invalid constant adjustment causing overflow/underflow or incorrect predicate flipping

### Strategy 3: Non-power-of-two types

- **Target**: Test with non-standard integer types (e.g., i24, i48) and illegal target types
- **Rationale**: shouldChangeType may incorrectly allow/disallow transformations for non-power-of-two or illegal types, especially with vector types.
- **Expected Issue**: Transformation to illegal type or missed optimization opportunity

### Strategy 4: Mixed signedness with same constant

- **Target**: Test both signed and unsigned strict predicates with same constant value that has different semantics
- **Rationale**: Constant adjustment logic is the same for signed/unsigned but overflow boundaries differ (INT_MIN vs 0, INT_MAX vs UINT_MAX).
- **Expected Issue**: Incorrect predicate flipping for signed comparisons near INT_MIN/INT_MAX

### Strategy 5: Multi-use shl with different predicates

- **Target**: Test shl with multiple icmp users with different predicates/constants
- **Rationale**: hasOneUse check prevents transformation but other optimizations might create patterns where transformation becomes valid for one use but not others.
- **Expected Issue**: Missed optimization or incorrect transformation when shl has multiple icmp users
