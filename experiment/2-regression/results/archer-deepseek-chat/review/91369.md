## Test Strategies

### Strategy 1: ZeroExtLargeConstant

- **Target**: Create test with stride value that has ZExtInst users and constant > 2^63
- **Rationale**: getSExtValue() will overflow or assert for large constants with ZExtInst, should use getZExtValue() for zero extension semantics
- **Expected Issue**: Incorrect constant value or assertion failure when converting large APInt to int64_t

### Strategy 2: ExtensionChain

- **Target**: Test stride -> trunc -> sext/zext chains
- **Rationale**: Patch only handles direct sext/zext users, misses indirect extensions through truncate or other operations
- **Expected Issue**: Missing constant propagation through multi-step extension chains

### Strategy 3: VectorExtension

- **Target**: Test with vectorized sext/zext operations
- **Rationale**: U->getType() for vector extensions returns vector type, ConstantInt::get expects integer type
- **Expected Issue**: Type mismatch or compilation error when creating constant for vector type

### Strategy 4: MixedSignedness

- **Target**: Stride with both SExtInst and ZExtInst users
- **Rationale**: Using getSExtValue() for all extensions ignores signedness differences between SExtInst and ZExtInst
- **Expected Issue**: Incorrect constant values for ZExtInst users when stride is negative

### Strategy 5: NonIntegerExtension

- **Target**: Test with pointer extensions or other non-integer types
- **Rationale**: ConstantInt::get expects integer type, but extensions could involve pointer types or other non-integer types
- **Expected Issue**: Compilation error or incorrect code generation for non-integer extension types
