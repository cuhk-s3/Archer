## Test Strategies

### Strategy 1: Multi-Operand PHI Recursion

- **Target**: PHI nodes with 3+ operands forming complex cycles
- **Rationale**: The fix only handles 2-operand PHIs. PHIs with 3+ operands could have multiple self-references or form more complex cycles that aren't broken.
- **Expected Issue**: Infinite recursion or incorrect known bits due to unhandled recursive patterns

### Strategy 2: Nested Recursion Chains

- **Target**: Chains of PHIs/selects longer than 2 levels
- **Rationale**: The fix looks one level deep. Longer chains (phi1->phi2->phi3->phi1) won't be fully unrolled, potentially missing recursion.
- **Expected Issue**: Analysis gets stuck or returns conservative results missing optimization opportunities

### Strategy 3: Min/Max Intrinsic Recursion

- **Target**: Min/max intrinsics (llvm.smax, llvm.umin, etc.) forming recursive patterns
- **Rationale**: TODO comment mentions min/max not handled. These could create similar recursive patterns as selects.
- **Expected Issue**: Infinite recursion or incorrect analysis when PHI appears in min/max intrinsic operands

### Strategy 4: Vector PHI Lane Recursion

- **Target**: Vector PHIs with per-lane recursive patterns
- **Rationale**: Vector operations might have different recursion patterns per lane. The fix treats vectors uniformly.
- **Expected Issue**: Incorrect known bits for vector lanes with different recursive relationships

### Strategy 5: Fast-Math Flag Interactions

- **Target**: PHI/select with fast-math flags in FP analysis
- **Rationale**: Fast-math flags affect NaN/zero semantics. Recursion breaking might propagate flags incorrectly.
- **Expected Issue**: Incorrect FP class analysis due to mishandled fast-math flags in recursive patterns

### Strategy 6: Context Instruction Edge Cases

- **Target**: Edge cases with null/invalid context instructions
- **Rationale**: Context instruction updates assume valid terminators. Unreachable blocks or unusual CFGs might break this.
- **Expected Issue**: Crash or incorrect analysis due to invalid context instruction

### Strategy 7: Poison/Undef in Recursive Patterns

- **Target**: Recursive patterns involving poison/undef values
- **Rationale**: Poison/undef semantics might interact unexpectedly with recursion breaking logic.
- **Expected Issue**: Incorrect propagation of poison/undef through recursive pattern analysis
