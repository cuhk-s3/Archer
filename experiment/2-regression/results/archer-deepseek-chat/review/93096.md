## Test Strategies

### Strategy 1: Wrapped Range Poison

- **Target**: Test with wrapped ConstantRange inputs to trunc
- **Rationale**: getActiveBits() uses getUnsignedMax() which for wrapped ranges returns APInt::getMaxValue, potentially overestimating active bits and incorrectly inferring nuw flag when truncation would lose non-zero bits from wrapped values.
- **Expected Issue**: Incorrect nuw flag inference leading to poison values at runtime

### Strategy 2: Empty Range Edge Case

- **Target**: Test with empty ConstantRange (isEmptySet() returns true)
- **Rationale**: Both getActiveBits() and getMinSignedBits() return 0 for empty sets, which would satisfy <= DestWidth condition, potentially adding flags to dead code or unreachable paths.
- **Expected Issue**: Flags added to unreachable code or incorrect optimization based on dead code analysis

### Strategy 3: Signed Boundary Precision

- **Target**: Test ranges at signed boundaries (e.g., [-128, 127] trunc to i8, [-129, 126] trunc to i8)
- **Rationale**: getMinSignedBits() uses getSignificantBits() which may have off-by-one errors at power-of-two boundaries, especially for ranges that include INT_MIN.
- **Expected Issue**: Incorrect nsw flag inference when truncation would change sign interpretation

### Strategy 4: Vector Element Uniformity

- **Target**: Test vector trunc with non-uniform per-element constraints
- **Rationale**: Scalar analysis using getScalarSizeInBits() assumes uniform behavior across vector elements, but different elements might have different range constraints.
- **Expected Issue**: Incorrect flag inference for some vector elements while correct for others

### Strategy 5: Range Widening Interaction

- **Target**: Test with SCCP range widening that creates over-approximated ranges
- **Rationale**: SCCP's widening heuristics might create ranges wider than actual possible values, causing getActiveBits()/getMinSignedBits() to overestimate and miss flag opportunities.
- **Expected Issue**: Missed optimization opportunities due to imprecise range analysis
