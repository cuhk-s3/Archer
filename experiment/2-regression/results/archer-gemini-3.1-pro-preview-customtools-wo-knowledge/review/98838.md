## Test Strategies

### Strategy 1: Insert PHINode before terminator in entry block

- **Target**: A test case that vectorizes a `PHINode` and causes it to be removed from its parent, so that it is processed by `~BoUpSLP()` without a parent.
- **Rationale**: The `~BoUpSLP()` destructor iterates over `DeletedInstructions` and for any instruction `I` that has no parent, it calls `I->insertBefore(F->getEntryBlock().getTerminator())`. If `I` is a `PHINode` (which can happen if a `PHINode` was vectorized and removed from its parent via `removeInstructionsAndOperands`), inserting it before the terminator of the entry block will place it after non-PHI instructions, violating LLVM IR invariants. This can trigger assertions in the `ilist` or `BasicBlock` implementation, or cause issues if the terminator is a special instruction like `CatchSwitchInst`.
- **Expected Issue**: The compiler will crash with an assertion failure or segmentation fault because `I->insertBefore(F->getEntryBlock().getTerminator())` is called on an instruction `I` that is a `PHINode`, which violates the LLVM IR invariant that `PHINode`s must be at the beginning of a basic block, or because `F->getEntryBlock().getTerminator()` is a `CatchSwitchInst` which must be the only non-PHI instruction in the block.
