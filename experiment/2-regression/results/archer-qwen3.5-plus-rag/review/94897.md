## Test Strategies

### Strategy 1: Test signed predicate flip with boundary constant

- **Target**: icmp slt/sgt with shl and constant at signed min/max boundary
- **Rationale**: The getFlippedStrictnessPredicateAndConstant function checks for overflow/underflow based on signedness. For signed predicates (slt/sgt), flipping strictness at boundary values (SMIN/SMAX) should fail, but we should verify the fix handles this correctly when combined with the trunc transformation.
- **Expected Issue**: The transformation might incorrectly apply when the constant is at signed boundary, leading to incorrect comparison results after truncation.

### Strategy 2: Test vector types with mixed undef elements

- **Target**: Vector icmp with shl where some elements have undef constants
- **Rationale**: The getFlippedStrictnessPredicateAndConstant function has special handling for vectors with undef elements (lines 6715-6726). The fix uses this function, so we should test if the vector handling works correctly when combined with the trunc transformation.
- **Expected Issue**: Incorrect handling of vector elements with undef values could lead to wrong predicate or constant being used after transformation.

### Strategy 3: Test scalable vector types

- **Target**: Scalable vector icmp with shl and constant requiring flip
- **Rationale**: The getFlippedStrictnessPredicateAndConstant has a separate code path for scalable vectors (lines 6727-6733). The fix should work correctly with scalable vectors, but this path may not be well tested.
- **Expected Issue**: The transformation might not correctly handle scalable vectors, potentially causing assertion failures or incorrect code generation.

### Strategy 4: Test nuw/nsw flag propagation with flipped predicate

- **Target**: shl with nuw/nsw flags combined with predicate flip transformation
- **Rationale**: The patch comment mentions 'nuw cannot be propagated as we always use ashr here'. The fix creates a trunc with the nsw flag from the original shl (line 2445). We should verify this is correct when the predicate is flipped.
- **Expected Issue**: The nsw flag on trunc might be incorrectly set when the predicate is flipped, potentially causing poison values in cases where the original code was well-defined.

### Strategy 5: Test equality predicates that should not be flipped

- **Target**: icmp eq/ne with shl and constant requiring flip (should not transform)
- **Rationale**: The fix checks ICmpInst::isStrictPredicate(CmpPred) before attempting the flip. Equality predicates (eq/ne) are not strict predicates and should not enter the flip logic. We should verify this check works correctly.
- **Expected Issue**: If the isStrictPredicate check fails for some edge case, equality predicates might incorrectly attempt to flip, causing assertion failures or incorrect transformations.
