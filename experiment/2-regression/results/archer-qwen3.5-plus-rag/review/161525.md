## Test Strategies

### Strategy 1: Test inf-inf with ordered predicates

- **Target**: fcmp ogt/olt/one (x - y), 0 where x=y=inf
- **Rationale**: The infinity check in the code only applies to UGT, ULT, UNE, OEQ, OGE, OLE predicates, but OGT, OLT, ONE predicates fall through without the same check. When x=y=+inf, x-y=nan, and fcmp ogt nan, 0 = false, but fcmp ogt +inf, +inf = false. This seems OK, but need to verify edge cases with mixed sign infinities.
- **Expected Issue**: Incorrect transformation when x and y are infinities of the same sign for OGT/OLT/ONE predicates, producing different results due to NaN from inf-inf

### Strategy 2: Test denormal mode non-IEEE

- **Target**: fcmp pred (x - y), 0 with preserve-sign or positive-zero denormal modes
- **Rationale**: The code requires DenormalMode::getIEEE() for the transformation. Test with non-IEEE denormal modes to verify the transformation is correctly blocked. With preserve-sign mode, denormal results from x-y could have different signs, affecting comparison with 0.
- **Expected Issue**: Transformation incorrectly applied or blocked depending on denormal mode handling

### Strategy 3: Test NaN inputs with unordered predicates

- **Target**: fcmp ueq/uge/ule (x - y), 0 where x or y is NaN
- **Rationale**: When x is NaN, x-y=NaN, and fcmp ueq NaN, 0 = true. But fcmp ueq NaN, y should also be true. Need to verify this holds for all unordered predicates and all combinations of NaN inputs.
- **Expected Issue**: Mismatch in unordered predicate semantics when NaN is involved

### Strategy 4: Test fast-math flags on fsub

- **Target**: fcmp pred (fsub nnan/ninf x, y), 0
- **Rationale**: The code checks for nnan/ninf flags on the fsub instruction. Test with various combinations of fast-math flags to ensure the transformation correctly handles them and propagates flags appropriately to the resulting fcmp.
- **Expected Issue**: Fast-math flags not properly considered or propagated, leading to incorrect optimization or missed optimization opportunities

### Strategy 5: Test vector types with mixed infinity/finite elements

- **Target**: fcmp pred (x - y), 0 with vector types where some elements are inf and others are finite
- **Rationale**: Vector operations process each element independently. Test with vectors where some lanes have inf-inf and others have finite values to ensure element-wise correctness of the transformation.
- **Expected Issue**: Incorrect transformation for vector types when infinity handling differs across vector lanes

### Analysis

### Bug Triggering Analysis
The patch adds a transformation for `fcmp pred (x - y), 0` to `fcmp pred x, y`. The code has special handling for predicates UGT, ULT, UNE, OEQ, OGE, OLE to check for infinity cases (inf - inf = nan), but OGT, OLT, ONE predicates fall through without the same explicit infinity check.

However, after extensive testing with Alive2 and difftest, the transformation appears to be semantically correct even for OGT/OLT/ONE predicates with infinity inputs:
- inf - inf = nan, fcmp ogt nan, 0 = false
- fcmp ogt inf, inf = false
- Both produce the same result

The transformation is correctly blocked for non-IEEE denormal modes (preserve-sign, positive-zero) as verified by Alive2 showing syntactic equality (no transformation).

### Fix Weakness Analysis
The fix appears to be correct. The key observations:
1. For OGT/OLT/ONE predicates, even though there's no explicit infinity check, the semantics work out correctly because both the original and transformed comparisons produce the same result for inf-inf cases.
2. The denormal mode check correctly prevents transformation for non-IEEE modes.
3. The nnan/ninf flags on fsub are correctly checked for the predicates that need them.
4. Vector types are handled correctly element-wise.

No bugs were found in the fix. The transformation is semantically correct for all tested cases.
