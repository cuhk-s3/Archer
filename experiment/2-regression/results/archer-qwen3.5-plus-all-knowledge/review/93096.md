## Test Strategies

### Strategy 1: Empty Range Test

- **Target**: trunc-nuw-nsw-flags.ll
- **Rationale**: Test the edge case where the operand range is empty (unreachable code). The fix sets both nsw and nuw flags when getActiveBits() and getMinSignedBits() return 0, which might be incorrect for unreachable values.
- **Expected Issue**: SCCP might incorrectly set nsw/nuw flags on trunc instructions in unreachable code blocks, potentially causing miscompilation if the flags are used for further optimizations.

### Strategy 2: Range Expansion After Flag Set

- **Target**: trunc-nuw-nsw-flags.ll
- **Rationale**: Test the scenario where the operand range expands after the nsw/nuw flag is set. The fix checks !hasNoUnsignedWrap() before re-evaluating the condition, so if the range expands after the flag is set, the flag won't be unset.
- **Expected Issue**: SCCP might set nsw/nuw flags based on incomplete range information, and fail to unset them when the range expands, leading to incorrect flags and potential miscompilation.

### Strategy 3: Sign-Wrapped Range Test

- **Target**: trunc-nuw-nsw-flags.ll
- **Rationale**: Test trunc with sign-wrapped ranges (e.g., [-4, 4) represented as [12, 4) in unsigned). For sign-wrapped ranges, getSignedMin() and getSignedMax() return conservative INT_MIN/INT_MAX values, which might cause getMinSignedBits() to return the full bit width.
- **Expected Issue**: SCCP might fail to set the nsw flag for sign-wrapped ranges that actually fit in the destination width, missing optimization opportunities.

### Strategy 4: Boundary Value Test

- **Target**: trunc-nuw-nsw-flags.ll
- **Rationale**: Test trunc with ranges at the boundary of the destination width. For example, trunc i8 to i4 with range [0, 16) should set nuw (max 15 fits in 4 bits), but range [0, 17) should not (max 16 doesn't fit).
- **Expected Issue**: Off-by-one errors in the condition (getActiveBits() <= DestWidth vs getActiveBits() < DestWidth) could cause incorrect flag setting at boundary values.

### Strategy 5: Mixed Sign Range Test

- **Target**: trunc-nuw-nsw-flags.ll
- **Rationale**: Test trunc with ranges that span both positive and negative values. For example, range [-8, 8) should set nsw (fits in 4 bits signed) but not nuw (negative values don't fit in 4 bits unsigned).
- **Expected Issue**: The fix might incorrectly set both flags or neither flag for mixed-sign ranges, depending on how getActiveBits() and getMinSignedBits() handle such ranges.
