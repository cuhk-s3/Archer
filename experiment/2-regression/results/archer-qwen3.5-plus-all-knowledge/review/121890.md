## Test Strategies

### Strategy 1: Test nusw-only GEP with unsigned comparison

- **Target**: icmp-gep.ll test cases with nusw flag
- **Rationale**: The CanFold check allows folding with just nusw (no nuw). But nusw only guarantees no signed wrap, not unsigned wrap. When converting unsigned comparison to signed, if the offset causes unsigned wrap but not signed wrap, the transformation could be incorrect. Test with offsets that cause unsigned wrap but not signed wrap.
- **Expected Issue**: Incorrect transformation where (gep nusw Ptr, OFFSET) ult Ptr is folded to OFFSET slt 0, but OFFSET causes unsigned wrap leading to wrong result

### Strategy 2: Test mixed nuw/nusw flags between two GEPs

- **Target**: icmp-gep.ll test cases comparing two GEPs with different flags
- **Rationale**: When comparing two GEPs, flags are intersected. If one GEP has nuw and the other has nusw, intersection is empty. But if both have nusw (no nuw), intersection has nusw and folding proceeds with signed predicate. Test cases where both GEPs have nusw but offsets could have different signs or cause unsigned wrap.
- **Expected Issue**: Incorrect signed comparison when both GEPs have only nusw, leading to wrong results for offsets with different signs

### Strategy 3: Test samesign flag with potentially different-sign offsets

- **Target**: icmp-gep.ll test cases with nusw nuw flags
- **Rationale**: When GEP has both nuw and nusw, the resulting icmp gets samesign flag. This produces poison if operands have different signs. Test with offsets that could have different signs to verify samesign flag is correctly applied and doesn't cause incorrect poison.
- **Expected Issue**: Poison produced incorrectly when offsets have different signs but samesign flag assumes they don't

### Strategy 4: Test vector GEP comparisons with nuw/nusw flags

- **Target**: Vector GEP test cases in icmp-gep.ll
- **Rationale**: Vector GEPs might have per-element semantics. The current code treats flags uniformly for all elements. Test vector GEP comparisons where different lanes might have different offset characteristics.
- **Expected Issue**: Incorrect folding for vector GEPs where per-element flag semantics differ

### Strategy 5: Test equality comparison with no nowrap flags

- **Target**: icmp-gep.ll equality comparison test cases without flags
- **Rationale**: Equality comparisons always fold regardless of flags. Verify this is correct by testing edge cases where offsets could wrap but equality should still hold.
- **Expected Issue**: Potential incorrect equality folding when offsets wrap in ways that affect pointer equality
