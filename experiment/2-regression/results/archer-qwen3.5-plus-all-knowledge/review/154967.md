## Test Strategies

### Strategy 1: Empty mask for FirstActiveLane

- **Target**: Early exit loop where the early exit condition is never true in any lane
- **Rationale**: When FirstActiveLane receives an all-zero mask, cttz returns the vector width. With interleaving, this could exceed the total lanes across parts, causing out-of-bounds extraction or incorrect selects in ExtractLane.
- **Expected Issue**: ExtractLane may extract from wrong part or with invalid index, leading to incorrect exit value or undefined behavior

### Strategy 2: Interleaved early exit with live-out induction variable

- **Target**: Early exit loop with interleaving (UF > 1) where the induction variable is used after the loop
- **Rationale**: The ExtractLane must correctly extract the induction value from the first active lane across all interleaved parts. If the lane calculation is off by one part or the comparison logic fails, the wrong IV value is returned.
- **Expected Issue**: Wrong induction variable value returned at loop exit, causing incorrect program behavior

### Strategy 3: Poison propagation in early exit condition

- **Target**: Early exit loop where the condition can produce poison in some lanes (e.g., invalid shift, out-of-bounds access in masked lanes)
- **Rationale**: If the early exit condition produces poison in any lane, FirstActiveLane may return poison or an incorrect lane index. ExtractLane then uses this poison index, propagating undefined behavior.
- **Expected Issue**: Poison propagates through ExtractLane, causing undefined behavior or miscompilation in the exit block

### Strategy 4: Unroll combined with interleaving for early exit

- **Target**: Early exit loop with both unroll factor > 1 and interleave factor > 1
- **Rationale**: VPlanUnroll.cpp adds operands to ExtractLane for each unrolled part. With both UF > 1 and IC > 1, the operand count and indexing logic may become incorrect, leading to wrong extraction.
- **Expected Issue**: ExtractLane has incorrect number of operands after unroll, extracting from wrong vector or with wrong index

### Strategy 5: Scalable vector early exit with interleaving

- **Target**: Early exit loop compiled for scalable vectors (vscale) with interleaving
- **Rationale**: The ExtractLane code uses RuntimeVF which should handle scalable vectors, but the comparison and subtraction logic assumes fixed-width semantics. With vscale, the lane index calculation may be incorrect.
- **Expected Issue**: Incorrect lane-to-part mapping for scalable vectors, extracting wrong value at early exit
