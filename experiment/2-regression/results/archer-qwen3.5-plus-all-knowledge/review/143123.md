## Test Strategies

### Strategy 1: Test FMF propagation from select with nsz flag

- **Target**: matchDecomposedSelectPattern with select having nsz but fcmp without nsz
- **Rationale**: The fix passes FMF from select instead of fcmp. We should test the case where select has nsz but fcmp doesn't - this should now allow min/max pattern matching that was previously blocked.
- **Expected Issue**: If the fix incorrectly handles FMF merging, the pattern might not be recognized when select has nsz but fcmp doesn't, or vice versa.

### Strategy 2: Test FMF propagation with nnan on fcmp but not on select

- **Target**: matchDecomposedSelectPattern where fcmp has nnan but select doesn't
- **Rationale**: The fix adds nnan from fcmp to the FMF passed to matchSelectPattern only if fcmp.hasNoNaNs() is true. We need to verify this doesn't incorrectly allow NaN-related optimizations when select doesn't have nnan.
- **Expected Issue**: If nnan from fcmp is incorrectly propagated when select doesn't have nnan, NaN handling optimizations might be incorrectly applied.

### Strategy 3: Test mismatched zeros with different FMF combinations

- **Target**: Select pattern with mismatched zeros (+0.0 vs -0.0) and various FMF combinations on select and fcmp
- **Rationale**: Issue 57357 shows that mismatched zeros with strict inequality and no nsz on select should not be optimized. The fix should preserve this behavior. We need to test all combinations of nsz on select/fcmp.
- **Expected Issue**: If FMF is not correctly propagated, mismatched zero patterns might be incorrectly optimized when select lacks nsz.

### Strategy 4: Test vector types with mixed FMF

- **Target**: Vector select/fcmp patterns where FMF might behave differently per element
- **Rationale**: Vector operations may have different FMF semantics. The fix uses isa<FPMathOperator>(SI) to check if select has FMF, but vector types might have element-wise FMF considerations.
- **Expected Issue**: Vector patterns might be incorrectly optimized if FMF propagation doesn't account for vector semantics.

### Strategy 5: Test cast operations with FMF propagation

- **Target**: matchDecomposedSelectPattern with CastOp parameter and FMF from select
- **Rationale**: The fix affects the code path where CastOp is used (lines 8921-8939). When FPToSI or FPToUI casts are involved, FMF.setNoSignedZeros() is called. We need to verify this interacts correctly with the new FMF parameter.
- **Expected Issue**: The interaction between cast-related FMF modifications and the new FMF parameter might cause incorrect pattern matching for cast patterns.
