## Test Strategies

### Strategy 1: Simplify `and` with `shl nuw` user

- **Target**: Create a test with an `and` instruction whose result is used by a `shl nuw` instruction. The `and` mask should cover all demanded bits of the `and` instruction itself, but the `shl` should demand all bits of its result.
- **Rationale**: The fix simplifies `and` instructions when the demanded bits are a subset of the `and` mask. However, it only clears assumptions (like `nuw`/`nsw`) on users whose demanded bits are not all ones. If a user like `shl nuw` demands all bits, its `nuw` flag is not cleared. The `and` might have been preventing the `shl` from shifting out non-zero bits (which would cause poison due to `nuw`). Removing the `and` exposes these non-zero bits to the `shl`, causing it to produce poison incorrectly.
- **Expected Issue**: The `and` instruction is simplified to its first operand, but the `shl` instruction retains its `nuw` flag. If the first operand has non-zero bits in the upper 8 bits, the `shl nuw` will produce poison, whereas the original `and` would have cleared those bits, making the `shl nuw` safe. This leads to a miscompilation where a valid value becomes poison.

### Strategy 2: Simplify `or` with `add nsw` user

- **Target**: Create a test with an `or` instruction whose result is used by an `add nsw` instruction. The `or` mask should not intersect with the demanded bits of the `or` instruction.
- **Rationale**: Similar to the `and` case, simplifying an `or` instruction might change the value fed into an `add nsw` instruction. If the `add nsw` demands all bits, its `nsw` flag is not cleared. Removing the `or` could change the sign of the operand, causing the `add` to undergo signed overflow and produce poison.
- **Expected Issue**: The `or` instruction is simplified to its first operand, but the `add` instruction retains its `nuw` flag. If the first operand has zero bits where the `or` mask had ones, the `add nuw` might now overflow, whereas the original `or` would have set those bits, potentially preventing overflow (or vice versa, depending on the exact values). Wait, `or` adds bits. If we remove `or`, the value becomes smaller, so `add nuw` is less likely to overflow. What about `nsw`? If we remove `or`, the sign bit might change.
