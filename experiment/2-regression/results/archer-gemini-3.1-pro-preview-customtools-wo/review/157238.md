## Test Strategies

### Strategy 1: Exact Positive Infinity

- **Target**: `bitcast i32 0x7f800000 to float`
- **Rationale**: The known bits exactly represent `+Inf` with a known sign bit (0). The code will fail to match `Bits == InfKB` because `InfKB` has an unknown sign bit.
- **Expected Issue**: The analysis will fail to infer `fcPosInf` and will instead return a broader class (e.g., `fcPosNormal | fcPosSubnormal | fcPosInf`).

### Strategy 2: Exact Negative Infinity

- **Target**: `bitcast i32 0xff800000 to float`
- **Rationale**: The known bits exactly represent `-Inf` with a known sign bit (1). The code will fail to match `Bits == InfKB` because `InfKB` has an unknown sign bit.
- **Expected Issue**: The analysis will fail to infer `fcNegInf` and will instead return a broader class.

### Strategy 3: Exact Positive Zero

- **Target**: `bitcast i32 0x00000000 to float`
- **Rationale**: The known bits exactly represent `+0.0` with a known sign bit (0). The code will fail to match `Bits == ZeroKB` because `ZeroKB` has an unknown sign bit.
- **Expected Issue**: The analysis will fail to infer `fcPosZero` and will instead return a broader class.

### Strategy 4: Exact Negative Zero

- **Target**: `bitcast i32 0x80000000 to float`
- **Rationale**: The known bits exactly represent `-0.0` with a known sign bit (1). The code will fail to match `Bits == ZeroKB` because `ZeroKB` has an unknown sign bit.
- **Expected Issue**: The analysis will fail to infer `fcNegZero` and will instead return a broader class.

### Strategy 5: Infinity with Unknown Sign Bit

- **Target**: `bitcast i32 (or (and X, 0x80000000), 0x7f800000) to float`
- **Rationale**: This tests the case where the sign bit is actually unknown, which should successfully match `Bits == InfKB`.
- **Expected Issue**: This should correctly infer `fcInf` (both positive and negative), serving as a baseline to show that the logic only works when the sign bit is unknown.
