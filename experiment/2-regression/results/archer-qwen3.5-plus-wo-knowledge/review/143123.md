## Test Strategies

### Strategy 1: Test FMF mismatch between select and fcmp for nsz flag

- **Target**: Create test where select has nsz but fcmp does not (or vice versa)
- **Rationale**: The fix changes FMF source from fcmp to select. If select has nsz but fcmp doesn't, the old code would incorrectly not recognize the pattern. Conversely, if fcmp has nsz but select doesn't, the old code might incorrectly recognize the pattern. The new code should use select's nsz flag.
- **Expected Issue**: Incorrect min/max pattern recognition when nsz flags differ between select and fcmp. May cause incorrect transformation or missed optimization.

### Strategy 2: Test FMF mismatch for nnan flag interaction

- **Target**: Create test where select has nnan but fcmp does not (or vice versa)
- **Rationale**: The fix merges nnan from fcmp into the FMF passed from select. This is done at line 8913-8914: `if (isa<FPMathOperator>(CmpI) && CmpI->hasNoNaNs()) FMF.setNoNaNs();`. Need to verify this merging is correct and doesn't cause issues when select has nnan but fcmp doesn't.
- **Expected Issue**: Incorrect NaN behavior determination when nnan flags differ. May cause unsafe transformations when fcmp lacks nnan but select has it, or missed optimizations when fcmp has nnan but select doesn't.

### Strategy 3: Test signed zero handling with mismatched flags

- **Target**: Create test with 0.0 vs -0.0 comparisons where select and fcmp have different nsz flags
- **Rationale**: Lines 8626 and 8743 check `!FMF.noSignedZeros()` to determine if signed zero handling is needed. The fix changes which FMF is used. Need to test cases where the sign of zero matters and flags differ between select and fcmp.
- **Expected Issue**: Incorrect handling of signed zeros leading to semantic changes. For example, minNum(0.0, -0.0) may return either value per IEEE 754, but fcmp/select has deterministic behavior.

### Strategy 4: Test vector types with per-element FMF differences

- **Target**: Create test with vector operations where FMF flags might behave differently
- **Rationale**: Vector operations may have different semantics for FMF flags. The fix should handle vectors correctly. Need to test cases where vector elements might have different NaN or signed zero behaviors.
- **Expected Issue**: Incorrect pattern recognition for vector types, potentially causing incorrect SIMD transformations.

### Strategy 5: Test edge case where fcmp is not FPMathOperator

- **Target**: Create test where CmpI is an integer comparison but select is floating point (type mismatch scenario)
- **Rationale**: The code at line 8903 checks `isa<FPMathOperator>(SI)` to get FMF from select. But the code at line 8913 checks `isa<FPMathOperator>(CmpI)`. Need to test scenarios where these might differ, especially in the cast handling code paths (lines 8921-8939).
- **Expected Issue**: Potential crash or incorrect behavior when dealing with type mismatches between fcmp operands and select values, especially with FPToSI/FPToUI casts.
