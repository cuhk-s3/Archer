## Test Strategies

### Strategy 1: Empty Range Test

- **Target**: trunc instruction in unreachable code
- **Rationale**: When the range is empty (isEmptySet()), both getActiveBits() and getMinSignedBits() return 0. The condition 0 <= DestWidth is always true, which would set both nsw and nuw flags. While this is technically safe for unreachable code, it could expose issues if the empty range handling is incorrect.
- **Expected Issue**: Incorrectly setting nsw/nuw flags on trunc from empty range, potentially causing issues if the code becomes reachable or if downstream passes make incorrect assumptions.

### Strategy 2: Wrapped Range Boundary Test

- **Target**: trunc with wrapped constant range
- **Rationale**: For wrapped ranges like [0xFFFFFF00, 0x00000100), getActiveBits() returns the active bits of the unsigned max (near 2^32-1), which should be 32. This should prevent nuw from being set. Similarly, getMinSignedBits() should correctly handle the signed min/max of wrapped ranges.
- **Expected Issue**: Incorrectly setting nuw/nsw flags on trunc when the range wraps around, leading to incorrect poison semantics.

### Strategy 3: Exact Boundary Test for NUW

- **Target**: trunc where getActiveBits() equals DestWidth exactly
- **Rationale**: Test cases where Range.getActiveBits() == DestWidth exactly (e.g., range [0, 255] truncated to i8). The condition uses <=, so this should set nuw. Need to verify this is correct for all boundary cases.
- **Expected Issue**: Off-by-one error in the boundary condition, either missing valid nuw opportunities or incorrectly setting nuw when the max value doesn't fit.

### Strategy 4: Exact Boundary Test for NSW

- **Target**: trunc where getMinSignedBits() equals DestWidth exactly
- **Rationale**: Test cases where Range.getMinSignedBits() == DestWidth exactly (e.g., range [-128, 127] truncated to i8). The condition uses <=, so this should set nsw. Need to verify this handles the sign extension correctly at boundaries.
- **Expected Issue**: Off-by-one error in the boundary condition for signed values, particularly for ranges that include values at the edge of representable signed values.

### Strategy 5: Vector Type Trunc Test

- **Target**: trunc instructions with vector types
- **Rationale**: The code uses getScalarSizeInBits() for DestWidth and GetRange should handle vectors by getting scalar range. Need to verify that vector trunc instructions are handled correctly, especially when different elements might have different ranges.
- **Expected Issue**: Incorrect handling of vector types, where the range analysis might not correctly account for per-element ranges or the scalar size calculation might be wrong.
