## Test Strategies

### Strategy 1: Test GEP chain with mixed nowrap flags

- **Target**: Create a chain of GEPs where some have inbounds/nuw/nusw flags and others don't
- **Rationale**: The fix intersects NoWrap flags across all GEPs in the chain. If flags are incorrectly computed or propagated, the transformation could be unsound. For example, if an outer GEP has nuw but an inner GEP doesn't, the combined flags might be wrong.
- **Expected Issue**: Incorrect comparison result due to wrong nowrap flag propagation, potentially leading to miscompilation when overflow behavior differs

### Strategy 2: Test GEP chain with 3+ levels

- **Target**: Create a deeply nested GEP chain (gep (gep (gep ptr, x), y), z) compared to base
- **Rationale**: The fix extends support from single GEP to chains. The EmitGEPOffsets function iterates in reverse order and sums offsets. Edge cases may exist with 3+ levels where the order of operations or flag handling differs from the single-GEP case.
- **Expected Issue**: Incorrect offset computation or flag handling for deep chains, causing wrong comparison results

### Strategy 3: Test GEP chain with vector types

- **Target**: Create GEP chains that produce vector of pointers compared to base pointer
- **Rationale**: The EmitGEPOffsets function has special handling for vector types (CreateVectorSplat). The interaction between vector GEP chains and the new CommonPointerBase logic may have edge cases, especially with mixed scalar/vector indices.
- **Expected Issue**: Incorrect vector splatting or type mismatch when computing offsets for vector GEP chains

### Strategy 4: Test GEP chain where RHS is also a GEP but not in LHS chain

- **Target**: Create case where LHS is a GEP chain but RHS is a different GEP that doesn't share the common base properly
- **Rationale**: The CommonPointerBase::compute function checks if RHS is in the LHS pointer chain. If RHS is a GEP but not part of the LHS chain, the function should return Base.Ptr=nullptr. However, edge cases in the chain walking logic could cause incorrect base detection.
- **Expected Issue**: Incorrect transformation when RHS is a GEP that partially overlaps with LHS chain but isn't the true base

### Strategy 5: Test GEP chain with zero indices

- **Target**: Create GEP chains that include zero indices (gep ptr, 0, x) compared to base
- **Rationale**: Zero-index GEPs are semantically no-ops but may affect the chain walking in CommonPointerBase::compute. The fix should handle these correctly, but there could be edge cases where zero indices affect flag computation or offset calculation.
- **Expected Issue**: Incorrect handling of zero-index GEPs in the chain, potentially causing wrong offset computation or flag intersection
