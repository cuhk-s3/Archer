## Test Strategies

### Strategy 1: Test ult comparison with nuw-only GEP vs base pointer

- **Target**: Add test case for icmp ult with getelementptr nuw compared to base pointer
- **Rationale**: The existing tests cover ugt and uge with nuw, but not ult or ule. This tests the case where nuw-only GEP is compared with ult predicate to its base pointer, which should transform to signed comparison (slt) since nuw alone doesn't guarantee signed non-wrap.
- **Expected Issue**: The transformation might incorrectly use unsigned comparison instead of signed, or might not handle the edge case where offset could be negative (but nuw still valid due to base pointer being small).

### Strategy 2: Test ule comparison with nuw-only GEP vs base pointer

- **Target**: Add test case for icmp ule with getelementptr nuw compared to base pointer
- **Rationale**: Similar to above, ule predicate with nuw-only GEP is not explicitly tested. The transformation should convert ule to sle since nuw alone doesn't have nusw.
- **Expected Issue**: Incorrect predicate conversion or missing samesign handling.

### Strategy 3: Test GEPs differing by one index with nuw-only flags

- **Target**: Add test case where two GEPs differ by exactly one operand index, both with nuw but not nusw
- **Rationale**: The patch handles three cases: GEP vs base, GEPs differing by one index, and general GEP vs GEP. While general GEP vs GEP with nuw is tested (test60_nuw), the 'differ by one index' case with nuw-only is not explicitly tested.
- **Expected Issue**: The transformation at line 835-838 might not correctly handle nuw-only case, potentially missing the signed predicate conversion.

### Strategy 4: Test vector GEP with nuw/nusw flags in icmp comparison

- **Target**: Add test case with vector of pointers GEP having nuw/nusw flags compared with unsigned predicate
- **Rationale**: The existing tests don't cover vector GEPs with nowrap flags. The samesign attribute on vector icmp requires each element pair to have the same sign, and this interaction with vector GEP nowrap flags is untested.
- **Expected Issue**: The setSameSign call might not work correctly for vector types, or the transformation might not properly handle element-wise nowrap flag semantics.

### Strategy 5: Test mixed nuw/nusw flags between two GEPs for unsigned comparison

- **Target**: Add test where one GEP has nuw only and the other has nusw only, with unsigned icmp
- **Rationale**: The test test60_nusw_nuw_mix shows that when flags don't match (nuw vs nusw), no transformation occurs. But the case where NW = (nuw & nusw) = none() should be tested more thoroughly, especially for the 'differ by one index' path.
- **Expected Issue**: The AND operation on flags at line 807 might produce unexpected results, or the CanFold check might incorrectly allow transformation when it shouldn't.
