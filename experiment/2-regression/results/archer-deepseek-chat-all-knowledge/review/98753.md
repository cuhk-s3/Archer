## Test Strategies

### Strategy 1: PointerProvenanceSubstitution

- **Target**: Test and/or with pointer equality and memory operations
- **Rationale**: Pointers with same address but different provenance should not be substituted in memory operations as it changes provenance information
- **Expected Issue**: Incorrect alias analysis or memory access after substitution leading to miscompilation

### Strategy 2: VectorCrossLaneSubstitution

- **Target**: Test vector equality with bitcasts/shuffles that change lane semantics
- **Rationale**: Per-lane equality doesn't guarantee equivalence for cross-lane operations like bitcasts that change lane count or shuffles that rearrange elements
- **Expected Issue**: Incorrect simplification when equality is only per-lane but operation is cross-lane

### Strategy 3: PoisonEqualitySubstitution

- **Target**: Test equality with poison/undef values in vectors and scalars
- **Rationale**: x == poison is poison, not false, and substitution might incorrectly assume equality or propagate poison incorrectly
- **Expected Issue**: Missing poison or incorrect poison propagation in simplification

### Strategy 4: IsConstantAssumption

- **Target**: Test llvm.is.constant with equality assumptions to constants
- **Rationale**: Equality with a constant value doesn't make a variable compile-time constant for llvm.is.constant intrinsic
- **Expected Issue**: Incorrect folding of llvm.is.constant based on equality assumption

### Strategy 5: RecursiveComplexPatterns

- **Target**: Test nested patterns and complex expressions that might cause issues with simplification depth or ordering
- **Rationale**: Complex patterns with multiple equalities or nested operations might cause infinite recursion or incorrect multi-step simplification
- **Expected Issue**: Infinite recursion or incorrect chained simplification
