## Test Strategies

### Strategy 1: MixedPoisonUndefFreeze

- **Target**: Create freeze instructions with mixed poison/undef patterns across lanes and verify vectorization preserves semantics
- **Rationale**: Tests Assumption 2 - vector freeze should have same semantics as element-wise scalar freezes. Mixed poison/undef patterns could expose differences in how 'arbitrary but fixed' values are chosen.
- **Expected Issue**: Vectorized code might produce different results than scalar code due to different freeze behavior for poison vs undef lanes

### Strategy 2: FreezeWithBitwidthReduction

- **Target**: Combine freeze with operations that trigger minimum bitwidth analysis and truncation
- **Rationale**: Tests Assumption 5 - freeze might interact incorrectly with bitwidth reduction. Truncating poison values before/after freeze could change semantics.
- **Expected Issue**: Incorrect poison propagation when freeze is combined with type demotion/truncation

### Strategy 3: FreezeMetadataPropagation

- **Target**: Add metadata (noundef, range, etc.) to freeze instructions and check if preserved after vectorization
- **Rationale**: Tests Assumption 4 - freeze doesn't call propagateMetadata unlike other instructions. Missing metadata could affect later optimizations.
- **Expected Issue**: Loss of important metadata attributes on vectorized freeze instructions

### Strategy 4: FreezeInComplexShuffles

- **Target**: Place freeze instructions in patterns requiring reordering or complex shuffle masks
- **Rationale**: Tests Assumption 6 - freeze might not handle shuffle/reordering correctly. Complex patterns could expose issues with mask generation.
- **Expected Issue**: Incorrect element ordering or poison propagation in shuffled freeze operations

### Strategy 5: FreezeCostModel

- **Target**: Create patterns where freeze cost significantly differs from CommonCost assumption
- **Rationale**: Tests Assumption 7 - CommonCost might not reflect actual freeze cost. Expensive operands or target-specific costs could affect profitability decisions.
- **Expected Issue**: Incorrect profitability analysis leading to suboptimal vectorization decisions
