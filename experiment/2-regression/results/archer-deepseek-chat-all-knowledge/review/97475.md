## Test Strategies

### Strategy 1: Poison in Condition Vector Elements

- **Target**: Create vector select with poison elements in condition and test if optimization incorrectly propagates poison
- **Rationale**: The optimization uses the condition to infer KnownBits but may not properly handle poison elements in vector conditions, potentially introducing unsound transformations.
- **Expected Issue**: Optimization may incorrectly fold select arm to constant when condition has poison elements, making result more poisonous than original.

### Strategy 2: Affected Values with Poison-Generating Flags

- **Target**: Create select where affected value has nsw/nuw flags and condition implies values that would violate those flags
- **Rationale**: KnownBits analysis may deduce constant values that would violate poison-generating flags on instructions, but optimization may replace with constant without dropping flags.
- **Expected Issue**: Replacing instruction with constant may lose poison-generating flags or incorrectly preserve them, changing poison semantics.

### Strategy 3: PHI Node Recursion Depth Bypass

- **Target**: Create deep chain of PHI nodes affected by condition to test recursion depth handling
- **Rationale**: Special depth handling for PHINodes (MaxAnalysisRecursionDepth - 2) may allow deeper recursion than intended or cut off analysis incorrectly.
- **Expected Issue**: May miss affected values through PHI chains or incorrectly analyze them due to depth limits.

### Strategy 4: Undef in Vector Constant Folding

- **Target**: Create vector select where KnownBits analysis produces constant with undef elements
- **Rationale**: When KnownBits becomes constant for vector type, it may have undef bits. Replacing select arm with such constant may introduce undef where none existed.
- **Expected Issue**: Vector constant with undef elements replaces non-undef value, making result more undefined.

### Strategy 5: Conflicting with Earlier Select Folds

- **Target**: Create pattern where this optimization enables unsafe subsequent transformation
- **Rationale**: This optimization runs late and creates constants that may enable other unsafe folds (e.g., removing guards for poison-generating operations).
- **Expected Issue**: Cascade of optimizations leads to incorrect poison propagation or removal of necessary guards.
