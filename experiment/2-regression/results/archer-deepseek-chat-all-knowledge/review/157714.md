## Test Strategies

### Strategy 1: Mixed flag GEP chain folding

- **Target**: Create test with GEP chain mixing inbounds/nuw/nusw flags and compare to base
- **Rationale**: The intersection logic in CommonPointerBase::compute might incorrectly compute flags, leading to incorrect poison semantics or missed optimizations
- **Expected Issue**: Incorrect flag intersection causing either missed optimization (too restrictive) or incorrect poison propagation (not restrictive enough)

### Strategy 2: Vector GEP chain with scalar base

- **Target**: Create test with vector GEPs in chain starting from scalar pointer
- **Rationale**: Vector GEP handling might not properly handle type conversions or offset computations when mixing scalar and vector types
- **Expected Issue**: Type mismatch or incorrect offset computation leading to miscompilation

### Strategy 3: Negative offset GEP chain

- **Target**: Create test with GEP chain containing negative offsets and compare to base
- **Rationale**: Offset summation might incorrectly handle signedness or overflow for negative offsets
- **Expected Issue**: Incorrect offset computation or overflow handling for negative indices

### Strategy 4: Mixed address space GEP chain

- **Target**: Create test with GEPs across different address spaces using addrspacecast
- **Rationale**: CommonPointerBase::compute might not properly handle address space casts or different sized address spaces
- **Expected Issue**: Incorrect base pointer identification or offset computation across address spaces

### Strategy 5: Poison-generating flag preservation

- **Target**: Create test with GEP chain where intermediate GEP has poison-generating flags that shouldn't propagate
- **Rationale**: Intersection of flags might incorrectly preserve poison-generating flags like samesign when they shouldn't apply to the entire chain
- **Expected Issue**: Incorrect poison propagation making transformed code more poisonous than original
