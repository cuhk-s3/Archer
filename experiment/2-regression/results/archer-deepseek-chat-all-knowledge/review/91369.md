## Test Strategies

### Strategy 1: ZExtNegativeStride

- **Target**: Test when stride is negative and used through ZExtInst
- **Rationale**: getSExtValue() will preserve negative sign, but ZExtInst expects zero extension. This could create incorrect constant (sign-extended negative value instead of zero-extended).
- **Expected Issue**: Incorrect constant value for zero-extended negative stride.

### Strategy 2: CastChainPropagation

- **Target**: Test stride used through multiple casts (e.g., sext->trunc->zext)
- **Rationale**: Current fix only handles direct SExtInst/ZExtInst users, not chains or other cast types.
- **Expected Issue**: Stride not properly constant-folded through cast chains.

### Strategy 3: ExitBlockStrideUse

- **Target**: Test stride used in loop exit block (LCSSA phi)
- **Rationale**: The fix assumes stride users have VPValues in the plan, but exit block uses might not.
- **Expected Issue**: Missing VPValue for exit block use leads to no replacement.

### Strategy 4: LargeStrideOverflow

- **Target**: Test stride value that overflows when sign-extended
- **Rationale**: getSExtValue() on large APInt may overflow int64_t, causing undefined behavior.
- **Expected Issue**: Overflow in constant creation or incorrect constant value.

### Strategy 5: VectorStrideExtension

- **Target**: Test stride with vector type extensions
- **Rationale**: Current code assumes scalar integer types for ConstantInt::get.
- **Expected Issue**: Type mismatch or assertion failure when creating constant.

### Strategy 6: StrideArithmeticBeforeExtension

- **Target**: Test stride used in arithmetic before extension (e.g., sext(stride + 1))
- **Rationale**: Fix only checks direct users of stride, not transitive uses.
- **Expected Issue**: Stride arithmetic not constant-folded.

### Strategy 7: MixedExtensionTypes

- **Target**: Test stride used through both SExtInst and ZExtInst in same loop
- **Rationale**: Need to handle both extension types correctly with appropriate extension method.
- **Expected Issue**: Incorrect constant for one extension type.
