## Test Strategies

### Strategy 1: PoisonAlternativeValue

- **Target**: Alternative value V in select/phi patterns
- **Rationale**: If the alternative value V contains poison elements or is poison itself, analyzing it instead of the recursive PHI could incorrectly propagate poison knowledge or miss that the select/phi shields the poison.
- **Expected Issue**: Incorrect known bits/FP classes when alternative value is poison but select/phi condition prevents poison propagation.

### Strategy 2: VectorSelectWithPartialPoison

- **Target**: Vector select instructions with poison in some lanes
- **Rationale**: Vector selects can have poison in individual lanes. The match pattern doesn't distinguish scalar vs vector, and analyzing the alternative vector value could incorrectly assume all lanes are like the non-PHI operand.
- **Expected Issue**: Incorrect vector lane analysis when select has mixed poison/non-poison lanes.

### Strategy 3: FastMathFlagsOnFPSelect

- **Target**: Floating-point select with fast-math flags
- **Rationale**: Fast-math flags (nnan, nsz, etc.) on FP selects affect NaN/zero behavior. The fix doesn't consider these flags when analyzing the alternative operand.
- **Expected Issue**: Incorrect FP class analysis for selects with fast-math flags that relax semantics.

### Strategy 4: MultiOperandPhiWithTwoUniqueValues

- **Target**: PHI with 3+ operands but only 2 unique values
- **Rationale**: The fix only handles 2-operand PHIs, but TODO mentions could handle any number if only two unique values. Missing this case could cause infinite recursion or missed optimization.
- **Expected Issue**: Analysis gets stuck or gives up on PHIs with >2 operands but simple value pattern.

### Strategy 5: NestedSelectPattern

- **Target**: Nested select patterns (select within select)
- **Rationale**: The fix only handles one level of select(PHI, V). Nested patterns like select(cond1, select(cond2, PHI, V1), V2) aren't handled and could cause infinite recursion.
- **Expected Issue**: Infinite recursion or incorrect analysis for nested select patterns involving PHI.

### Strategy 6: MinMaxIntrinsicPattern

- **Target**: llvm.min.*/llvm.max.* intrinsics with PHI operand
- **Rationale**: TODO mentions Min/Max not handled. These intrinsics are semantically similar to selects and common in optimizations.
- **Expected Issue**: Missed optimization opportunities or incorrect analysis for min/max patterns with PHI.

### Strategy 7: ConditionDependentAnalysis

- **Target**: Select condition providing additional known bits
- **Rationale**: The fix ignores the select condition which might provide known bits about the PHI (e.g., select(icmp eq PHI, 0, PHI, 1)).
- **Expected Issue**: Missing known bits that could be derived from the select condition.

### Strategy 8: ContextInstructionMismatch

- **Target**: Context instruction update for complex CFG
- **Rationale**: Setting CxtI to incoming block terminator might not be correct for all analysis queries, especially with multiple levels of indirection.
- **Expected Issue**: Incorrect context-sensitive analysis (assumptions, dominator info) due to wrong CxtI.

6. **ContextInstructionMismatch**: Complex CFGs with nested loops cause issues with context instruction handling.

7. **FastMathFlagsOnFPSelect**: Fast-math flags on FP selects aren't considered.
