## Test Strategies

### Strategy 1: Mixed GEP Flags Intersection

- **Target**: Test intersection logic when LHS and RHS GEPs have different nowrap flags
- **Rationale**: The patch uses NW = GEPLHS->getNoWrapFlags() & GEPRHS->getNoWrapFlags(). If LHS has nuw and RHS has nusw, intersection is none(), preventing fold. But CanFold checks NW != none(), so fold won't happen. Need to verify this is correct semantics.
- **Expected Issue**: Incorrectly preventing fold when it should be allowed, or allowing fold when it shouldn't

### Strategy 2: Negative Index with NUW

- **Target**: GEP with nuw flag and negative index (non-inbounds)
- **Rationale**: nuw guarantees no unsigned wrap, but negative index produces negative offset. Folding icmp ult to icmp ult on negative offset may give wrong result compared to pointer comparison.
- **Expected Issue**: Incorrect comparison result when offset is negative

### Strategy 3: Vector GEP with Poison Elements

- **Target**: Vector GEP with nuw/nusw and poison elements in indices
- **Rationale**: Vector GEPs with poison elements may produce poison results. The fold may incorrectly propagate poison or drop poison semantics.
- **Expected Issue**: Incorrect poison propagation in vector case

### Strategy 4: Zero-Sized Element Type

- **Target**: GEP with nowrap flags indexing zero-sized type
- **Rationale**: Offset is always 0 regardless of index. The fold should produce constant comparison, but need to verify nowrap flags semantics with zero-sized types.
- **Expected Issue**: Incorrect handling of zero-sized types with nowrap flags

### Strategy 5: Type Conversion in EmitGEPOffset

- **Target**: GEP with index type different from pointer index type
- **Rationale**: EmitGEPOffset may compute offset in different integer type. Type conversions may affect comparison semantics, especially with nowrap flags.
- **Expected Issue**: Incorrect offset comparison due to type conversion issues

### Strategy 6: SameSign Flag Validity

- **Target**: GEP with nusw nuw flags producing offset that may not have same sign as 0
- **Rationale**: samesign flag asserts both operands have same sign. Need to verify offset from nusw nuw GEP always has same sign as 0 for the comparison.
- **Expected Issue**: Incorrect samesign flag application leading to poison

### Strategy 7: Large Offsets Near Wrap Boundary

- **Target**: GEP with nuw and index near unsigned wrap boundary
- **Rationale**: nuw prevents wrap, but offset near boundary may have different signed vs unsigned comparison behavior.
- **Expected Issue**: Incorrect fold near wrap boundaries
