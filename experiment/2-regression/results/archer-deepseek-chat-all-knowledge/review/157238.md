## Test Strategies

### Strategy 1: NonIEEEFormatAnalysis

- **Target**: Test bitcast to bfloat16, fp8, fp8e4m3fn formats
- **Rationale**: The patch only handles IEEE formats via Ty->isIEEE() check. Non-IEEE formats have different NaN/infinity encodings and should either be handled differently or skipped.
- **Expected Issue**: Incorrect NaN/Inf/Zero detection for non-IEEE formats, or missed optimization opportunities.

### Strategy 2: VectorElementWiseMismatch

- **Target**: Test m_ElementWiseBitCast with unusual vector types (scalable vectors, different element counts)
- **Rationale**: The pattern matcher might not correctly identify all element-wise bitcasts, causing the analysis to skip valid cases.
- **Expected Issue**: Analysis incorrectly skipped for valid element-wise bitcasts, missing optimization opportunities.

### Strategy 3: ComplexIntegerSource

- **Target**: Test bitcast from complex integer expressions with recursion depth limits
- **Rationale**: computeKnownBits has limited recursion depth. Complex integer sources might not yield complete known bits, leading to incomplete FPClass analysis.
- **Expected Issue**: Incomplete FPClass information due to truncated known bits analysis.

### Strategy 4: SignBitSpecialValues

- **Target**: Test bitcast patterns where sign bit matters for infinity/zero detection
- **Rationale**: The code clears sign bit in InfKB.Zero and ZeroKB.Zero before comparison, which might incorrectly match +Inf with -Inf bit patterns or +0 with -0.
- **Expected Issue**: Incorrect sign classification for infinity/zero values.

### Strategy 5: PoisonUndefPropagation

- **Target**: Test bitcast from integer sources with poison/undef bits
- **Rationale**: KnownBits might not fully capture poison semantics. Poison bits in integer could affect float classification incorrectly.
- **Expected Issue**: Incorrect FPClass analysis when integer source contains poison/undef values.
