## Test Strategies

### Strategy 1: FMF Mismatch - Select without nsz, fcmp with nsz

- **Target**: Floating-point select pattern with fcmp having nsz flag but select lacking it
- **Rationale**: The fix changes FMF source from fcmp to select. If fcmp has nsz (no-signed-zeros) but select doesn't, the old code would incorrectly allow transformations that could change zero sign. Test should verify the transformation is now blocked when select lacks nsz.
- **Expected Issue**: Without the fix, the optimizer might incorrectly fold fcmp/select to minnum/maxnum when select lacks nsz, potentially changing -0.0 to +0.0 or vice versa. With the fix, such transformations should be blocked.

### Strategy 2: FMF Mismatch - Select with nnan, fcmp without nnan

- **Target**: Floating-point select pattern where select has nnan but fcmp doesn't
- **Rationale**: The fix passes FMF from select to matchDecomposedSelectPattern, but also adds nnan from fcmp if CmpI->hasNoNaNs(). Test should verify correct behavior when only select has nnan flag.
- **Expected Issue**: The transformation behavior should depend on select's FMF, not fcmp's. If select has nnan, NaN-related optimizations should be allowed regardless of fcmp's flags.

### Strategy 3: Vector FMF with mixed flags

- **Target**: Vector floating-point operations where FMF might differ between select and fcmp
- **Rationale**: Vector operations can have complex FMF scenarios. Test should verify that the fix correctly handles vector types where select and fcmp have different FMF combinations.
- **Expected Issue**: Incorrect FMF propagation could lead to wrong minnum/maxnum conversions for vectors, potentially producing different NaN or signed-zero behavior than the original select.

### Strategy 4: Select with no FMF, fcmp with nnan only

- **Target**: Pattern where fcmp has nnan but select has no FMF flags
- **Rationale**: The fix adds nnan from fcmp if hasNoNaNs() is true, even when select has no FMF. This tests the specific code path at line 8913-8914 where FMF.setNoNaNs() is called conditionally.
- **Expected Issue**: The nnan flag from fcmp should be preserved for NaN behavior analysis, but other flags like nsz should not be inherited. Test should verify nnan is correctly used for NaN behavior determination while nsz-dependent transformations are blocked.

### Strategy 5: Select with full FMF, fcmp with no FMF

- **Target**: Pattern where select has comprehensive FMF (nnan, nsz, ninf) but fcmp has none
- **Rationale**: This tests the reverse scenario - when select is more permissive than fcmp. The fix should allow optimizations based on select's FMF even when fcmp lacks flags.
- **Expected Issue**: With the fix, optimizations that require nnan/nsz should proceed based on select's flags. Without the fix, the code would incorrectly use fcmp's empty FMF and block valid optimizations.
