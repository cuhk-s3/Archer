## Test Strategies

### Strategy 1: Test nuw-only GEP with unsigned comparison

- **Target**: gep nuw with icmp ult/ugt/ule/uge comparing two GEPs with same base
- **Rationale**: The fix allows folding unsigned icmp of GEPs with only nuw flag (without nusw). However, nuw only guarantees no unsigned wrap, not that the signed interpretation is preserved. When comparing offsets, if the offsets have different signs, the unsigned comparison of pointers may not equal the unsigned comparison of offsets. For example, if offset1 = -1 (large unsigned) and offset2 = 1, pointer comparison (base-1 <u base+1) depends on whether base-1 wraps, but offset comparison (-1 <u 1) = false while pointer comparison could be true if no wrap occurs.
- **Expected Issue**: Incorrect transformation: (gep nuw ptr, -1) ult (gep nuw ptr, 1) folded to (-1) ult 1 = false, but the original pointer comparison could be true if the addresses don't wrap

### Strategy 2: Test nuw-only GEP comparing with base pointer

- **Target**: gep nuw compared directly with base pointer using unsigned predicate
- **Rationale**: Similar to above, when folding (gep nuw ptr, offset) cmp ptr to (offset cmp 0), if the GEP only has nuw (not nusw), negative offsets are allowed as long as they don't wrap unsigned. A negative offset like -1 with nuw would fold to (-1) ugt 0 = true, but the actual pointer comparison depends on the address space layout.
- **Expected Issue**: Incorrect transformation: (gep nuw ptr, -1) ugt ptr folded to (-1) ugt 0 = true, but the pointer comparison result depends on actual address values

### Strategy 3: Test mixed nuw/nusw flags on two GEPs

- **Target**: Two GEPs with different nowrap flag combinations (one nuw only, one nusw only)
- **Rationale**: When comparing two GEPs, the code intersects their flags at line 807. If LHS has nuw and RHS has nusw, the intersection is none(), which should prevent folding. But we should test edge cases where the intersection results in only nuw or only nusw to verify the CanFold logic handles these correctly.
- **Expected Issue**: Potential incorrect folding when flag intersection results in only nuw without nusw for unsigned comparisons

### Strategy 4: Test vector GEPs with nuw flags

- **Target**: Vector of pointers GEP with nuw flags in unsigned icmp
- **Rationale**: The fix doesn't appear to have special handling for vector types in the new lambdas. Vector GEPs with nuw flags might have per-element nowrap semantics that could interact incorrectly with the folding logic, especially if some lanes have different sign patterns.
- **Expected Issue**: Incorrect transformation for vector GEPs where per-element nowrap flags don't uniformly guarantee the offset comparison equivalence

### Strategy 5: Test samesign attribute propagation

- **Target**: GEP with nusw+nuw flags producing icmp with samesign attribute
- **Rationale**: When both nusw and nuw are present, the code sets samesign on the resulting icmp. We need to verify this is correct and that the samesign attribute properly constrains the result to poison if operands have different signs. Test cases where offsets could theoretically have different signs despite the nowrap flags would expose issues.
- **Expected Issue**: Missing poison generation when samesign icmp receives operands of different signs, or incorrect samesign attribute application
