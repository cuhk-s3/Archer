## Test Strategies

### Strategy 1: MixedNoWrapChain

- **Target**: Create GEP chains with mixed no-wrap flags (nuw/nusw/inbounds) to test flag accumulation logic
- **Rationale**: The AND operation on no-wrap flags may clear all flags even when individual GEPs have flags, potentially breaking the CanFold check for unsigned comparisons.
- **Expected Issue**: Incorrect transformation or missed optimization when accumulated flags become empty.

### Strategy 2: OffsetOverflowChain

- **Target**: Create chains where intermediate offset computation overflows index type despite individual GEPs having no-wrap flags
- **Rationale**: EmitGEPOffsets sums scaled offsets; multiplication by type size could overflow even if GEP has nuw/nusw for the index itself.
- **Expected Issue**: Incorrect offset computation leading to wrong comparison result.

### Strategy 3: VectorGEPChain

- **Target**: Test chains of vector GEPs with vector indices and mixed vector/scalar operations
- **Rationale**: Vector GEP handling may have edge cases in offset computation and type conversion.
- **Expected Issue**: Miscompilation with vector types or incorrect offset scaling.

### Strategy 4: ZeroSizedTypeChain

- **Target**: Create GEP chains involving zero-sized types at different levels
- **Rationale**: Zero-sized types produce zero offset regardless of index, which may break offset accumulation logic.
- **Expected Issue**: Incorrect offset when zero-sized types are involved in the chain.

### Strategy 5: AddressSpaceCrossing

- **Target**: Test GEP chains that cross address spaces via addrspacecast
- **Rationale**: CommonPointerBase may not properly handle addrspacecast in the chain, leading to incorrect base identification.
- **Expected Issue**: Missed optimization or incorrect transformation when address spaces differ.

This is because the GEP with nuw flag and index 2305843009213693951 multiplied by 8 overflows 64 bits, making the GEP poison.
