## Test Strategies

### Strategy 1: Multi-use indirect propagation

- **Target**: Test cases where ctlz/cttz appears to have single use but the shift result has multiple users
- **Rationale**: If the shift result is used elsewhere, making the shift amount poison could poison those uses unexpectedly
- **Expected Issue**: Unexpected poison propagation to users of shift result

### Strategy 2: Vector element-wise poison

- **Target**: Vector types with mixed zero/non-zero elements
- **Rationale**: For vector ctlz/cttz, some elements might be zero (become poison) while others not, affecting shift results element-wise
- **Expected Issue**: Incorrect poison propagation in vector elements

### Strategy 3: Metadata/attribute invalidation

- **Target**: ctlz/cttz with noundef or other UB-implying attributes
- **Rationale**: Changing zero_is_poison to true might invalidate noundef attribute which should be dropped
- **Expected Issue**: Invalid metadata/attributes after transformation

### Strategy 4: Control flow dependent shifts

- **Target**: Shifts in conditional branches or loops
- **Rationale**: Poison might propagate differently through control flow, especially if shift is conditionally executed
- **Expected Issue**: Incorrect poison semantics across control flow

### Strategy 5: Shift flag interactions

- **Target**: Shifts with nuw/nsw/exact flags
- **Rationale**: These flags might have special interactions with poison operands
- **Expected Issue**: Incorrect flag semantics with poison shift amounts

### Strategy 6: Non-canonical shift patterns

- **Target**: Patterns that match m_Shift but aren't simple shifts
- **Rationale**: The m_Shift pattern might match unexpected instructions or patterns
- **Expected Issue**: Incorrect transformation on non-shift instructions

## Bugs Found

### Bug #1

**Analysis:**
Found a bug! The transformation changes cttz from zero_is_poison=false to true, but doesn't drop the noundef attribute. When y=0, the transformed code returns poison which violates noundef, causing UB. The original code returns 32 (not poison), satisfying noundef. The fix should drop UB-implying attributes like noundef when changing zero_is_poison to true, similar to what's done in the similar patch from Phase 1 (Top1: Drop noundef in `foldSelectCttzCtlz`).

**Original LLVM IR:**
```llvm

define i32 @shl_cttz_false(i32 %x, i32 %y) {
entry:
  %cttz = call noundef i32 @llvm.cttz.i32(i32 %y, i1 false)
  %res = shl i32 %x, %cttz
  ret i32 %res
}

; Function Attrs: nocallback nofree nosync nounwind speculatable willreturn memory(none)
declare i32 @llvm.cttz.i32(i32, i1 immarg) #0

define i32 @main(i32 %argc, ptr %argv) {

  %r = call i32 @shl_cttz_false(i32 1, i32 0)
```
