## Test Strategies

### Strategy 1: Signed-negative values in unsigned comparisons

- **Target**: Create test where signed negative variable is used in unsigned comparison (e.g., %x = -1, then %x uge 0)
- **Rationale**: Tests if adding Var >= 0 constraint creates inconsistency with signed value semantics. Unsigned comparison of negative signed value wraps to large positive, but signed constraint x >= 0 would be false.
- **Expected Issue**: Constraint system inconsistency or missed optimization opportunity due to contradictory constraints.

### Strategy 2: Variable appears in both signed and unsigned systems

- **Target**: Variable used in both signed and unsigned comparisons in same function
- **Rationale**: Tests if the Var >= 0 constraint in unsigned system conflicts with possible signed negative constraints. The same variable could have x < 0 in signed system but x >= 0 in unsigned system.
- **Expected Issue**: Incorrect inference or constraint system contradiction when reasoning across signed/unsigned systems.

### Strategy 3: Large number of new variables in single fact

- **Target**: Constraint that introduces many new variables simultaneously
- **Rationale**: Tests performance impact and potential assertion failures. Each new variable adds a Var >= 0 constraint, potentially causing quadratic growth in constraint system size.
- **Expected Issue**: Performance degradation, assertion failure, or incorrect constraint count in debug assertions.

### Strategy 4: Variables with boundary values (0, UINT_MAX)

- **Target**: Variables with values at unsigned boundaries
- **Rationale**: Tests edge cases for Var >= 0 constraint. For x = 0, constraint is redundant. For x = UINT_MAX, signed interpretation might be negative.
- **Expected Issue**: Redundant constraints causing inefficiency or overflow in coefficient computation.

### Strategy 5: Duplicate variable introduction

- **Target**: Same variable introduced multiple times through different constraints
- **Rationale**: Tests handling of duplicate Var >= 0 constraints. The fix adds constraints for each NewVariable without checking if constraint already exists.
- **Expected Issue**: Duplicate constraints in system or assertion failure about constraint count mismatch.

### Strategy 6: Pointer arguments with unsigned comparisons

- **Target**: Function arguments that are pointers used in unsigned comparisons
- **Rationale**: Tests semantic correctness of Var >= 0 for pointers. While pointers can be compared unsigned, the non-negativity constraint might not be semantically valid for all pointers.
- **Expected Issue**: Incorrect optimization or semantic mismatch in constraint reasoning.

### Strategy 7: Interaction with equality constraints

- **Target**: Equality constraints involving newly introduced variables
- **Rationale**: Tests ordering of constraint addition. The fix adds Var >= 0 before handling equality inversion, which could affect constraint propagation.
- **Expected Issue**: Missing or incorrect constraint propagation for equality cases.
