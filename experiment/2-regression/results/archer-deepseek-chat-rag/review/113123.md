## Test Strategies

### Strategy 1: Special Constant Values

- **Target**: Constant values C in fcmp patterns
- **Rationale**: Test with special FP values (NaN, Inf, ±0.0, denormals) that may break bitwiseIsEqual or neg() assumptions. For example, neg(NaN) is NaN but bitwiseIsEqual may fail for different NaN encodings.
- **Expected Issue**: Incorrect transformation or missed optimization due to special constant handling

### Strategy 2: Fast-Math Flag Mismatch

- **Target**: Fast-math flags on individual fcmp instructions
- **Rationale**: Test with conflicting fast-math flags (e.g., one has nnan, other doesn't). The patch unions flags which may be unsafe if flags imply different semantics.
- **Expected Issue**: Unsafe optimization or incorrect NaN handling due to flag union

### Strategy 3: Vector Undef Patterns

- **Target**: Vector constant patterns with undef elements
- **Rationale**: Test non-splat vectors with mixed undef/constant elements. m_APFloatAllowUndef only handles splat undef, not general vector constant patterns.
- **Expected Issue**: Miscompilation or incorrect transformation for partial undef vectors

### Strategy 4: Boundary Value Semantics

- **Target**: Variable x values at boundaries (±C)
- **Rationale**: Test inclusive vs exclusive comparisons at boundary values. The transformation must preserve exact semantics for x = C and x = -C cases.
- **Expected Issue**: Incorrect results at boundary conditions due to predicate mismatch

### Strategy 5: Sign-Only Operation Variants

- **Target**: Variable operand with sign-only ops (fneg, fabs, copysign)
- **Rationale**: Test patterns like (fneg(x) < C) && (fneg(x) > -C) or with fabs. The TODO mentions generalizing to handle negated variable operand.
- **Expected Issue**: Missed optimization opportunities or incorrect transformation
