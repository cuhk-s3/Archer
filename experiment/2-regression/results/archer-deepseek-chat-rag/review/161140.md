## Test Strategies

### Strategy 1: PoisonFlagInterchange

- **Target**: Test conversions between instructions with different poison-generating flags (nuw/nsw/exact)
- **Rationale**: shl nsw x, 1 has different poison conditions than mul nsw x, 2. The patch doesn't preserve or adjust flags, potentially causing miscompilation.
- **Expected Issue**: Incorrect poison semantics after conversion leading to miscompilation

### Strategy 2: ConstantPositionMismatch

- **Target**: Test Sub/Shl/AShr with constant on LHS instead of RHS
- **Rationale**: isBinOpWithConstantInt only checks RHS for Sub/Shl/AShr. 1 << x is not convertible to x * 2 but might be incorrectly handled.
- **Expected Issue**: Incorrect conversion or assertion failure

### Strategy 3: TypeMismatchInterchange

- **Target**: Test conversions between different integer types or vector types
- **Rationale**: Code assumes type compatibility but doesn't verify. i8 << 1 to i8 * 2 vs i32 << 1 to i32 * 2 may have different overflow behavior.
- **Expected Issue**: Type mismatch in generated code or assertion failure

### Strategy 4: EdgeConstantValue

- **Target**: Test boundary constant values like shift >= bitwidth, all-ones, zero, etc.
- **Rationale**: shl x, 32 (i32) is always poison, but mul x, 2^32 wraps. Conversions may change semantics.
- **Expected Issue**: Different behavior after conversion leading to miscompilation

### Strategy 5: MultipleConversionChoices

- **Target**: Test VL with multiple interchangeable options (e.g., [x + 0, y * 1, z & -1])
- **Rationale**: Algorithm must choose consistent conversion. Choice may affect cost or validity of vectorization.
- **Expected Issue**: Inconsistent conversion or suboptimal vectorization

## Bugs Found

### Bug #1

**Analysis:**
Test EdgeConstantValue: Actually 2^32 = 4294967296 which is 0 mod 2^32. So mul i32 y, 0 = 0. shl i32 x, 32 is poison. Different semantics.

  ret i32 %add
}
