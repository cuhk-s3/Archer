## Test Strategies

### Strategy 1: Non-IEEE Format Edge Cases

- **Target**: Test bitcast from integer to non-IEEE float types (x86_fp80, PPC_FP128)
- **Rationale**: The patch only applies special NaN/Inf/Zero detection for IEEE formats. Non-IEEE formats have different bit layouts and semantics. Need to ensure sign bit transfer works correctly and special value detection doesn't incorrectly apply.
- **Expected Issue**: Incorrect fpclass inference for non-IEEE formats, or missed optimizations where sign bit information could still be transferred.

### Strategy 2: Sign Bit Clearing for Special Values

- **Target**: Test bitcast patterns where sign bit matters for zero/infinity detection
- **Rationale**: The code clears sign bit in InfKB.Zero and ZeroKB.Zero before comparison. This assumes sign bit doesn't affect equality, but +0/-0 and +Inf/-Inf have different bit patterns. Need to test cases where sign bit knowledge could improve precision.
- **Expected Issue**: Missing sign information for zero/infinity classifications, or incorrect handling of signed zeros/infinities.

### Strategy 3: Vector Element-Wise Bitcast Edge Cases

- **Target**: Test vector bitcasts with partial element demands and different vector configurations
- **Rationale**: The patch uses m_ElementWiseBitCast which requires same vector element count. Need to test edge cases like scalar-to-vector, vector-to-scalar via bitcast, and partial DemandedElts propagation through bitcast.
- **Expected Issue**: Incorrect handling of vector cases, missed optimizations for partial vector elements, or crashes on malformed bitcasts.

### Strategy 4: KnownBits Conflict Scenarios

- **Target**: Test cases where computeKnownBits returns conflicting information (hasConflict() == true)
- **Rationale**: If KnownBits has conflicts (Zero and One intersect), the analysis might produce incorrect results. The patch doesn't check for conflicts before using the bits.
- **Expected Issue**: Incorrect fpclass inference from conflicting bit information, potentially leading to miscompilation.

### Strategy 5: NaN Subclass Distinctions

- **Target**: Test bitcast patterns that could distinguish signaling vs quiet NaNs
- **Rationale**: The patch treats all NaNs as fcNan, but IEEE has signaling (sNaN) and quiet (qNaN) variants with different bit patterns. KnownBits might reveal enough information to distinguish them.
- **Expected Issue**: Missing precision in NaN classification, could distinguish sNaN vs qNaN based on quiet bit.

### Strategy 6: Depth Recursion and Complex Chains

- **Target**: Test deep chains of bitcasts and integer operations leading to the float bitcast
- **Rationale**: The patch uses Depth + 1 for computeKnownBits recursion. Complex integer expression chains might exceed depth limits or produce incomplete bit information.
- **Expected Issue**: Incomplete fpclass inference due to recursion limits or missing bit information from complex expressions.

### Strategy 7: Endianness-Sensitive Bitcasts

- **Target**: Test bitcasts involving vector types on big-endian vs little-endian targets
- **Rationale**: Bitcast semantics depend on endianness for vector-to-integer conversions. The patch might assume little-endian semantics.
- **Expected Issue**: Incorrect fpclass inference on big-endian targets due to misinterpreted bit positions.
