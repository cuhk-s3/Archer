## Test Strategies

### Strategy 1: ComplexSignAnalysisMutation

- **Target**: Test cases with values having complex sign patterns that ComputeNumSignBits may misanalyze
- **Rationale**: The patch relies on ComputeNumSignBits and MaskedValueIsZero to determine safe truncation widths. Complex sign patterns (e.g., values derived from select, phi, or complex arithmetic) may be misanalyzed, leading to unsafe truncation.
- **Expected Issue**: Incorrect truncation causing value corruption or undefined behavior when high bits are not actually zero/sign-extended.

### Strategy 2: DeepTruncationChainMutation

- **Target**: Test cases with chains of trunc/ext operations longer than depth limit (2-3)
- **Rationale**: The patch uses MaxDepthLevel limits to prevent deep recursion. Valid but deep truncation chains may be incorrectly rejected, missing optimization opportunities or causing inconsistent analysis.
- **Expected Issue**: Missed optimization opportunities or inconsistent minbitwidth decisions in different parts of the tree.

### Strategy 3: SignednessMisidentificationMutation

- **Target**: Test cases where signedness analysis may be incorrect for values with complex provenance
- **Rationale**: The patch determines signedness using isKnownNonNegative and opcode checks. Complex value flows (through phi, select, calls) may lead to incorrect signedness identification, causing wrong extension type (zext vs sext).
- **Expected Issue**: Incorrect sign extension/zero extension leading to value corruption in vectorized code.

### Strategy 4: ReductionMixedTypesMutation

- **Target**: Test cases with reduction operations involving mixed signed/unsigned types or poison values
- **Rationale**: Reduction bitwidth calculation uses ComputeNumSignBits and demanded bits. Mixed types or poison values may cause incorrect bitwidth calculation or signedness determination.
- **Expected Issue**: Incorrect reduction bitwidth leading to overflow/underflow or incorrect reduction results.

### Strategy 5: CastTrackingGapMutation

- **Target**: Test cases with BitCast, pointer casts, or vector casts not tracked in CastMaxMinBWSizes
- **Rationale**: CastMaxMinBWSizes only tracks ZExt, SExt, and Trunc. Other casts (BitCast, ptrtoint, etc.) may affect type sizes but aren't tracked, leading to incomplete analysis.
- **Expected Issue**: Incorrect minbitwidth analysis missing optimization opportunities or making unsafe assumptions.

### Strategy 6: ProfitabilityCacheInconsistencyMutation

- **Target**: Test cases where same value appears in different contexts with different profitability
- **Rationale**: AnalyzedMinBWVals caches values found unprofitable. If the same value appears in a different context where it would be profitable, the cache may prevent optimization.
- **Expected Issue**: Missed optimization opportunities due to overly aggressive caching.

### Strategy 7: TypeConversionSafetyMutation

- **Target**: Test cases with overflow/underflow risks in type narrowing conversions
- **Rationale**: The patch assumes CreateIntCast handles all safety concerns, but narrowing conversions may overflow/underflow if value ranges aren't properly constrained.
- **Expected Issue**: Overflow/underflow in narrowed vector operations causing incorrect results.
