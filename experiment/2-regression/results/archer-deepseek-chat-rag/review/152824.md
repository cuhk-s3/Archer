## Test Strategies

### Strategy 1: NaN Sign Bit Edge Cases

- **Target**: Test cases with known never-signaling-NaN but possible quiet-NaN with either sign
- **Rationale**: The code assumes when NaN is impossible, sign bit can be inferred from negative/positive knowledge. But 'never signaling NaN' still allows quiet NaN with either sign bit. Need to test cases where noNaNs flag is false but we have other constraints.
- **Expected Issue**: Incorrect sign bit inference when quiet NaN with opposite sign is possible

### Strategy 2: Vector Constant Mixed NaN Signs

- **Target**: Vector constants containing both positive and negative NaN elements
- **Rationale**: Vector constant handling ORs classify() results and checks sign consistency. classify() for NaN includes sign bit, so mixing positive and negative NaNs could cause incorrect sign bit inference.
- **Expected Issue**: Incorrect sign bit determination for vectors with mixed-sign NaN elements

### Strategy 3: maxnum/minnum Zero Sign Handling

- **Target**: maxnum/minnum with -0.0 and +0.0 operands under different fast math flags
- **Rationale**: IEEE 754 allows maxNum(-0.0, +0.0) to return either sign. The patch's sign inference logic at lines 4606-4616 needs to handle nsz flag and the uncertainty in zero sign results.
- **Expected Issue**: Incorrect sign bit inference for maxnum/minnum with mixed signed zeros

### Strategy 4: FMul NaN Sign Bit Ordering

- **Target**: FMul operations where 0*inf produces NaN, testing sign bit inference vs NaN detection order
- **Rationale**: Lines 4967-4972 infer sign bit before lines 4975-4978 check for 0*inf NaN. If sign is inferred then NaN is discovered, sign bit should be cleared but might not be.
- **Expected Issue**: Sign bit incorrectly retained when 0*inf produces NaN

### Strategy 5: Denormal Mode Sign Propagation

- **Target**: Operations with subnormal inputs under DAZ/FTZ modes
- **Rationale**: When denormals are flushed to zero, sign of subnormal affects sign of resulting zero. Sign bit inference needs to consider denormal flushing modes.
- **Expected Issue**: Incorrect sign bit inference when subnormals are flushed to signed zeros

### Strategy 6: Recursive Pattern Depth Limits

- **Target**: Deep chains of operations that trigger recursion depth limits
- **Rationale**: The old implementation had specific depth handling. New implementation might have different recursion patterns causing early termination or infinite recursion.
- **Expected Issue**: Missing sign bit inference due to recursion depth or excessive computation

### Strategy 7: FPExt/FPTrunc NaN Sign Preservation

- **Target**: Type conversions with NaN values and sign bit tracking
- **Rationale**: Lines 5087-5089 clear sign bit if NaN is possible after FPExt. But NaN sign bit is preserved in conversions. Need to test if sign bit handling is correct for NaN-preserving conversions.
- **Expected Issue**: Incorrect sign bit clearing for NaN values during type conversions
