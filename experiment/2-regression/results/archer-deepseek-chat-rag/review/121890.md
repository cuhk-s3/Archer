## Test Strategies

### Strategy 1: Test nuw with signed-negative offsets

- **Target**: GEP with nuw flag and offset that is negative when interpreted as signed
- **Rationale**: nuw ensures no unsigned wrap, but offset can be negative in signed interpretation. Folding ult to slt might be incorrect if pointer arithmetic wraps differently in signed vs unsigned domain.
- **Expected Issue**: Incorrect comparison result when offset is large positive (MSB set) but nuw prevents unsigned wrap.

### Strategy 2: Test nusw with unsigned wrap scenarios

- **Target**: GEP with nusw flag but offset that causes unsigned wrap
- **Rationale**: nusw prevents signed wrap in offset calculation, but unsigned wrap could still occur. The patch assumes nusw alone is sufficient for folding to signed comparison.
- **Expected Issue**: Incorrect folding when offset calculation stays within signed bounds but crosses unsigned boundary.

### Strategy 3: Test mixed nowrap flags intersection

- **Target**: GEP comparisons with different nowrap flag combinations
- **Rationale**: The patch uses flag intersection (NW = GEPLHS->getNoWrapFlags() & GEPRHS->getNoWrapFlags()). Need to test edge cases where one GEP has nuw, other has nusw, or other combinations.
- **Expected Issue**: Overly conservative or incorrect folding decisions for mixed flag combinations.

### Strategy 4: Test samesign flag propagation

- **Target**: Transforms that create or modify icmp instructions after samesign flag is set
- **Rationale**: The patch sets samesign flag when GEP has nusw nuw. Other passes might not preserve this flag, or might incorrectly propagate it.
- **Expected Issue**: Lost optimization opportunities or incorrect transformations due to missing samesign flag.

### Strategy 5: Test vector GEP with nowrap flags

- **Target**: Vector GEP comparisons with nowrap flags
- **Rationale**: Vector GEPs might have different semantics for nowrap flags. EmitGEPOffset and flag handling might not correctly handle vector cases.
- **Expected Issue**: Incorrect folding for vector GEP comparisons or miscompilation with vector nowrap semantics.
