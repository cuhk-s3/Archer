## Test Strategies

### Strategy 1: Negative value with shl nsw and icmp ult

- **Target**: A test case with `shl nsw` on a negative value, followed by `icmp ult` with a constant.
- **Rationale**: The patch incorrectly passes `Shl->hasNoSignedWrap()` as the `IsNUW` argument to `CreateTrunc`. If the input is negative (e.g., `-1`), `shl nsw` is valid, but `trunc nuw` will produce `poison` because the dropped bits are all ones, not zeros.
- **Expected Issue**: The `trunc` instruction will be generated with the `nuw` flag, which will evaluate to `poison` for negative inputs, leading to a miscompilation where a valid boolean result becomes `poison`.

### Strategy 2: Negative value with shl nsw and icmp ugt

- **Target**: A test case with `shl nsw` on a negative value, followed by `icmp ugt` with a constant.
- **Rationale**: Similar to Strategy 1, but using `icmp ugt` to ensure the bug is triggered across different unsigned predicates that are handled by the fold.
- **Expected Issue**: The `trunc` instruction will be generated with the `nuw` flag, which will evaluate to `poison` for negative inputs, leading to a miscompilation.

### Strategy 3: Vector of negative values with shl nsw

- **Target**: A test case with `shl nsw` on a vector of negative values, followed by a vector `icmp ult` or `icmp ugt`.
- **Rationale**: The bug should also manifest in vector types. Using a vector of negative values will verify that the incorrect flag propagation affects vector truncations as well.
- **Expected Issue**: The `trunc` instruction will be generated with the `nuw` flag, which will evaluate to `poison` for negative vector elements, leading to a miscompilation.

### Strategy 4: shl nsw with icmp ule (flipped strictness)

- **Target**: A test case with `shl nsw` on a negative value, followed by `icmp ule` with a constant that triggers the flipped strictness fold.
- **Rationale**: The patch specifically added support for flipped strictness predicates. Testing `icmp ule` ensures that the bug is triggered when the new flipped strictness logic is exercised.
- **Expected Issue**: The `trunc` instruction will be generated with the `nuw` flag, which will evaluate to `poison` for negative inputs, leading to a miscompilation.

### Strategy 5: shl nsw with icmp uge (flipped strictness)

- **Target**: A test case with `shl nsw` on a negative value, followed by `icmp uge` with a constant that triggers the flipped strictness fold.
- **Rationale**: Testing `icmp uge` ensures that the bug is triggered when the new flipped strictness logic is exercised for the greater-than-or-equal case.
- **Expected Issue**: The `trunc` instruction will be generated with the `nuw` flag, which will evaluate to `poison` for negative inputs, leading to a miscompilation.
