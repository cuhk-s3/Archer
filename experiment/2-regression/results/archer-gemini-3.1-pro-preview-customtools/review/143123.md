## Test Strategies

### Strategy 1: Missing nnan on select, present on fcmp

- **Target**: A select instruction without `nnan` flag, where its condition is an fcmp instruction with the `nnan` flag.
- **Rationale**: The fix changes `matchDecomposedSelectPattern` to use the FMF from the select instruction instead of the fcmp instruction. If the fcmp has `nnan` but the select does not, the previous behavior would incorrectly use `nnan` for the combined operation. The fix corrects this, so we should test that a select without `nnan` (but with an fcmp with `nnan`) is not optimized as if it had `nnan`.
- **Expected Issue**: The optimization might incorrectly fold the select into a min/max intrinsic or operation that propagates the nnan flag, even though the original select did not have the nnan flag. This could lead to incorrect behavior if the input is NaN.

### Strategy 2: Missing nsz on select, present on fcmp

- **Target**: A select instruction without `nsz` flag, where its condition is an fcmp instruction with the `nsz` flag.
- **Rationale**: Similar to `nnan`, if the fcmp has `nsz` but the select does not, the previous behavior would incorrectly use `nsz` for the combined operation. The fix corrects this, so we should test that a select without `nsz` (but with an fcmp with `nsz`) is not optimized as if it had `nsz`.
- **Expected Issue**: The optimization might incorrectly fold the select into a min/max intrinsic or operation that propagates the nsz flag, even though the original select did not have the nsz flag. This could lead to incorrect sign of zero.

### Strategy 3: nsz on fcmp is ignored if select lacks it

- **Target**: A select instruction without `nsz` flag, where its condition is an fcmp instruction with the `nsz` flag, and the optimization requires `nsz` (e.g., matching min/max with signed zeros).
- **Rationale**: The fix explicitly propagates `nnan` from fcmp to the FMF used for matching, but it does NOT propagate `nsz` or other flags from fcmp. Therefore, if fcmp has `nsz` but select does not, the `nsz` flag is not used for the match. This is a change in behavior. We should test that `nsz` on fcmp alone does not allow optimizations that require `nsz`.
- **Expected Issue**: The optimization might fail to fold the select into a min/max intrinsic or operation because it doesn't see the nnan flag on the fcmp, even though the select has it. Wait, the fix uses the select's FMF. If the select has `nnan` but fcmp doesn't, it will use `nnan`. But wait, the fix also adds `if (isa<FPMathOperator>(CmpI) && CmpI->hasNoNaNs()) FMF.setNoNaNs();`. So if fcmp has `nnan`, it is added to the FMF. So the FMF used is `SelectFMF | (CmpFMF.nnan)`. Let's test if `nsz` on fcmp is ignored if select doesn't have it.

### Strategy 4: Missing ninf on select, present on fcmp

- **Target**: A select instruction without `ninf` flag, where its condition is an fcmp instruction with the `ninf` flag.
- **Rationale**: If the fcmp has `ninf` but the select does not, the previous behavior would incorrectly use `ninf` for the combined operation. The fix corrects this, so we should test that a select without `ninf` (but with an fcmp with `ninf`) is not optimized as if it had `ninf`.
- **Expected Issue**: The optimization might incorrectly fold the select into a min/max intrinsic or operation that propagates the ninf flag, even though the original select did not have the ninf flag. This could lead to incorrect behavior if the input is infinity.

### Strategy 5: Select with nsz, fcmp without nsz

- **Target**: A select instruction with `nsz` flag, where its condition is an fcmp instruction without the `nsz` flag.
- **Rationale**: If the select has `nsz` but the fcmp does not, the new behavior will use the `nsz` from the select. The previous behavior used the FMF from the fcmp, so it would not see `nsz`. This means the fix enables optimizations when the select has `nsz` even if the fcmp does not. We should test that this optimization now triggers.
- **Expected Issue**: The optimization might incorrectly fold the select into a min/max intrinsic or operation that propagates the nsz flag, even though the original select did not have the nsz flag. This could lead to incorrect sign of zero.
