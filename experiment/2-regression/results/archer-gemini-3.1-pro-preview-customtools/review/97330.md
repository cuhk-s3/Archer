## Test Strategies

### Strategy 1: Select Condition Context

- **Target**: A test case with a `select` instruction where the condition is a complex expression that can be simplified by `SimplifyDemandedBits`.
- **Rationale**: The patch passes `SimplifyQuery` to `SimplifyDemandedBits`. If `SimplifyDemandedBits` is called on the condition of a `select` instruction, and the `SimplifyQuery` contains context from the `select` itself (e.g., assuming the condition is true in the true branch), it might incorrectly simplify the condition.
- **Expected Issue**: The `SimplifyDemandedBits` call on the `select` condition might incorrectly assume the condition is true or false based on the context of the `select` itself, leading to an incorrect simplification of the condition and potentially a miscompilation.

### Strategy 2: Branch Condition Context

- **Target**: A test case with a `br` instruction where the condition is a complex expression that can be simplified by `SimplifyDemandedBits`.
- **Rationale**: Similar to the `select` condition, if `SimplifyDemandedBits` is called on the condition of a `br` instruction, and the `SimplifyQuery` contains context from the `br` itself, it might incorrectly simplify the condition.
- **Expected Issue**: The `SimplifyDemandedBits` call on the `br` condition might incorrectly assume the condition is true or false based on the context of the `br` itself, leading to an incorrect simplification of the condition and potentially a miscompilation.

### Strategy 3: Multi-Use Instruction Context

- **Target**: A test case with an instruction that has multiple uses, where one use provides context that allows simplification, but the other use does not.
- **Rationale**: The patch passes `SimplifyQuery` to `SimplifyDemandedBits`. If an instruction has multiple uses, and `SimplifyDemandedBits` is called on it with a `SimplifyQuery` containing context from one of the uses, it might incorrectly simplify the instruction for all uses.
- **Expected Issue**: The `SimplifyDemandedBits` call on an instruction with multiple uses might incorrectly use context from one use to simplify the instruction for all uses, leading to a miscompilation.

### Strategy 4: Loop Context

- **Target**: A test case with an instruction inside a loop, where context from outside the loop allows simplification, but the simplification is not valid for all iterations of the loop.
- **Rationale**: The patch passes `SimplifyQuery` to `SimplifyDemandedBits`. If an instruction is inside a loop, and `SimplifyDemandedBits` is called on it with a `SimplifyQuery` containing context from outside the loop, it might incorrectly simplify the instruction.
- **Expected Issue**: The `SimplifyDemandedBits` call on an instruction inside a loop might incorrectly use context from outside the loop, leading to a miscompilation.

### Strategy 5: Poison-Generating Flags Context

- **Target**: A test case with an instruction that has poison-generating flags, where context allows dropping the flags, but the flags are actually needed.
- **Rationale**: The patch passes `SimplifyQuery` to `SimplifyDemandedBits`. If an instruction has poison-generating flags (e.g., `nsw`, `nuw`), and `SimplifyDemandedBits` is called on it with a `SimplifyQuery` containing context that allows dropping the flags, it might incorrectly drop the flags.
- **Expected Issue**: The `SimplifyDemandedBits` call on an instruction with poison-generating flags might incorrectly drop the flags based on context, leading to a miscompilation.
