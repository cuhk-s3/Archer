## Test Strategies

### Strategy 1: Signed comparison with trunc nuw

- **Target**: Test with `icmp slt (trunc nuw X), (trunc nuw Y)`
- **Rationale**: The patch allows folding `icmp (trunc nuw X), (trunc nuw Y)` for unsigned and equality comparisons. If the logic incorrectly allows it for signed comparisons, it would be a bug.
- **Expected Issue**: The fold might incorrectly apply to a signed comparison with trunc nuw, leading to incorrect results.

### Strategy 2: Unsigned comparison with mixed trunc nuw and trunc nsw

- **Target**: Test with `icmp ult (trunc nuw X), (trunc nsw Y)`
- **Rationale**: The patch requires both truncs to be nuw or both to be nsw for unsigned comparisons. If it incorrectly allows one nuw and one nsw, it would be a bug.
- **Expected Issue**: The fold might incorrectly apply to an unsigned comparison with one trunc nuw and one trunc nsw, leading to incorrect results.
