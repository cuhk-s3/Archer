### Strategy 1: Sign of NaN or Zero Ignored

- **Target**: Return instructions with `nofpclass` attributes where the sign of NaN or zero is not demanded but might be used by the caller.
- **Rationale**: The `SimplifyDemandedUseFPClass` function assumes that if a class is not in `DemandedMask`, it can be replaced with a constant or modified. However, if the caller demands a specific class but doesn't care about the sign of NaN or zero, the function might incorrectly change the sign of the returned value.
- **Expected Issue**: The function might return a constant that does not match the original value's sign, or it might return poison when it shouldn't, leading to incorrect behavior if the caller actually uses the sign of the NaN or zero.

- **Target**: Return instructions with `nofpclass` attributes where `DemandedMask` evaluates to `fcNone`.
- **Rationale**: The `SimplifyDemandedUseFPClass` function returns poison if `DemandedMask` is `fcNone`. However, if the original value was not poison and the caller expects a valid value, returning poison could lead to undefined behavior.
- **Expected Issue**: The function might return poison when it shouldn't, leading to undefined behavior.

- **Rationale**: The `SimplifyDemandedUseFPClass` function replaces a value with a constant if the demanded classes match a single bit pattern. However, if the original value was not a constant and the caller expects a dynamic value, replacing it with a constant could lead to incorrect behavior.
- **Expected Issue**: The function might incorrectly replace a value with a constant, leading to incorrect behavior.

- **Target**: Return instructions with `nofpclass` attributes that return the result of a `copysign` instruction.
- **Rationale**: The `SimplifyDemandedUseFPClass` function modifies the operands of instructions like `copysign` based on the demanded classes. However, if the modification is incorrect or if the caller expects the original operands, this could lead to incorrect behavior.
- **Expected Issue**: The function might incorrectly modify the operands of an instruction, leading to incorrect behavior.

- **Target**: Return instructions with `nofpclass` attributes that return the result of a `select` instruction.
- **Rationale**: The `SimplifyDemandedUseFPClass` function simplifies `select` instructions based on the demanded classes of their operands. However, if the simplification is incorrect or if the caller expects the original `select` instruction, this could lead to incorrect behavior.
- **Expected Issue**: The function might incorrectly simplify a `select` instruction, leading to incorrect behavior.
