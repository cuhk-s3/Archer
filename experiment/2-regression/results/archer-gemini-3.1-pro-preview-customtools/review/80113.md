## Test Strategies

### Strategy 1: Test missing clearAssumptionsOfUsers for non-all-ones demanded bits

- **Target**: `clearAssumptionsOfUsers` in `llvm/lib/Transforms/Scalar/BDCE.cpp`
- **Rationale**: The fix in `clearAssumptionsOfUsers` only adds an instruction `J` to the worklist if its demanded bits are not all ones (`!DB.getDemandedBits(J).isAllOnes()`). However, if `J`'s demanded bits are all ones, it is not added to the worklist, and its poison-generating flags are not dropped. This is incorrect because even if `J`'s demanded bits are all ones, its operands might have changed (e.g., a binary operator was simplified), which could invalidate its poison-generating flags (like `nsw`, `nuw`, `exact`).
- **Expected Issue**: The instruction `J` will not be added to the worklist, and its poison-generating flags will not be dropped. This can lead to incorrect optimizations later on, as the assumptions about the instruction's behavior are no longer valid.
