## Test Strategies

### Strategy 1: Context Instruction Propagation

- **Target**: SimplifyQuery.CxtI propagation through recursive calls
- **Rationale**: The fix changes how SimplifyQuery is passed through recursive calls, particularly the CxtI field. If the context instruction is not properly propagated or if it's incorrectly set to null in some paths, it could lead to incorrect known bits computation.
- **Expected Issue**: Incorrect known bits computation leading to wrong simplifications or missed optimization opportunities.

### Strategy 2: Depth Parameter Consistency

- **Target**: Depth parameter handling in recursive SimplifyDemandedBits calls
- **Rationale**: The fix adds Depth parameter to SimplifyDemandedBits but there might be inconsistencies in how Depth is incremented across different code paths, potentially causing infinite recursion or incorrect analysis depth limits.
- **Expected Issue**: Infinite recursion or incorrect analysis due to depth limit miscalculation.

### Strategy 3: SimplifyQuery Construction Edge Cases

- **Target**: SimplifyQuery.getWithInstruction() and getWithoutDomCondCache() usage
- **Rationale**: The fix introduces new SimplifyQuery construction patterns. Edge cases where getWithInstruction() is called with null or where getWithoutDomCondCache() interacts with other query fields could lead to incorrect context propagation.
- **Expected Issue**: Null pointer dereference or incorrect context propagation in known bits analysis.

### Strategy 4: Multiple Use Demanded Bits Context

- **Target**: SimplifyMultipleUseDemandedBits with new SimplifyQuery parameter
- **Rationale**: The fix changes SimplifyMultipleUseDemandedBits to accept SimplifyQuery instead of Instruction* CxtI. If the context instruction in the query doesn't match the expected caller context, it could lead to incorrect known bits from context computation.
- **Expected Issue**: Incorrect known bits computation for instructions with multiple uses.

### Strategy 5: Condition Context Integration

- **Target**: SimplifyQuery.CC (CondContext) field interaction with known bits
- **Rationale**: The commit message mentions enabling CondContext in the future. If CondContext is already being used somewhere or if there are latent bugs in how it interacts with known bits computation, the changes could expose them.
- **Expected Issue**: Incorrect known bits when conditional context is present.
