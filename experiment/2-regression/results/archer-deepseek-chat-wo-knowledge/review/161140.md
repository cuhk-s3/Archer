## Test Strategies

### Strategy 1: OverflowFlagInterchange

- **Target**: Mutate test cases to add nsw/nuw flags to operations that are being converted
- **Rationale**: Converting add nsw x, 0 to or x, 0 loses overflow semantics which could cause miscompilation when overflow flags are relied upon
- **Expected Issue**: Miscompilation when overflow flags are used by later instructions or for optimization

### Strategy 2: PoisonPropagationInterchange

- **Target**: Insert poison values as operands to interchangeable instructions
- **Rationale**: Different operations propagate poison differently (e.g., and poison, -1 vs mul poison, 1)
- **Expected Issue**: Incorrect poison propagation leading to undefined behavior

### Strategy 3: ConstantPositionInterchange

- **Target**: Swap constant operand positions in commutative operations
- **Rationale**: Code assumes constants are in specific positions for certain operations (right operand for sub, shl, ashr)
- **Expected Issue**: Failure to recognize interchangeable instructions due to constant position assumption violation

### Strategy 4: TypeBoundaryInterchange

- **Target**: Use constants at type boundaries (e.g., shift by type width-1)
- **Rationale**: shl x, 31 vs mul x, 2147483648 for i32 may have different overflow behavior at type boundaries
- **Expected Issue**: Incorrect overflow/underflow behavior or poison value generation

### Strategy 5: MixedAlternateInterchange

- **Target**: Create sequences with both alternate opcodes and interchangeable instructions
- **Rationale**: Complex interaction between alternate opcode feature and new interchangeable instruction feature may have unhandled cases
- **Expected Issue**: Incorrect instruction selection or operand transformation leading to wrong results

### Strategy 6: CostModelInterchange

- **Target**: Create cases where interchangeable conversion changes architecture-specific costs
- **Rationale**: SLPVectorizer uses cost models; wrong conversion could lead to suboptimal vectorization decisions
- **Expected Issue**: Performance regression or missed optimization opportunities

### Strategy 7: UnsupportedOpInterchange

- **Target**: Try to mix supported operations with unsupported ones (e.g., sdiv)
- **Rationale**: Edge cases in operation filtering logic when encountering unsupported operations in VL
- **Expected Issue**: Crash or incorrect vectorization of unsupported operations

### Strategy 8: MultiUseConstantInterchange

- **Target**: Constants used in multiple interchangeable operations
- **Rationale**: Constant sharing across multiple instructions might affect transformation decisions
- **Expected Issue**: Incorrect operand transformation due to shared constant analysis

### Strategy 9: NegativeConstantInterchange

- **Target**: Use negative constants with add/sub transformations
- **Rationale**: add x, -1 vs sub x, 1 transformation logic needs to handle negative constants correctly
- **Expected Issue**: Incorrect constant negation or transformation leading to wrong results

### Strategy 10: FloatTypeInterchange

- **Target**: Attempt to use floating point types with integer operations
- **Rationale**: Code assumes ConstantInt but might get confused with constant floats or wrong type handling
- **Expected Issue**: Type system violation, crash, or incorrect type casting
