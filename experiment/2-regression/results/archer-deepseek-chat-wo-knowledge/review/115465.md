## Test Strategies

### Strategy 1: Select with non-constant RHS

- **Target**: shufflevector(select(c, x, y), C) transformation
- **Rationale**: The fix only checks if RHS is constant, but doesn't verify that the shuffle mask is safe to distribute across the select. If the shuffle mask contains poison elements or selects from both operands in ways that interact with select semantics, the transformation could be incorrect.
- **Expected Issue**: Incorrect poison propagation or miscompilation when shuffle mask contains poison elements or selects from both vector operands in ways that don't commute with select.

### Strategy 2: PHI with non-dominating operands

- **Target**: shufflevector(phi(x, y), C) transformation
- **Rationale**: The foldOpIntoPhi function has complex logic for phi-translatable operands. If shufflevector has operands that aren't phi-translatable or don't dominate the phi block, the transformation could be incorrect.
- **Expected Issue**: Miscompilation when shufflevector has operands that can't be safely phi-translated to predecessor blocks.

### Strategy 3: Scalable vectors edge case

- **Target**: Both select and phi transformations
- **Rationale**: The fix checks match(RHS, m_Constant()) but doesn't explicitly exclude scalable vectors. The surrounding code has checks for scalable vectors in other transformations. Scalable vectors with constant RHS might have different semantics.
- **Expected Issue**: Incorrect transformation for scalable vectors where constant shuffle masks have different semantics.

### Strategy 4: Multiple uses of select/phi

- **Target**: FoldOpIntoSelect and foldOpIntoPhi usage
- **Rationale**: FoldOpIntoSelect has a FoldWithMultiUse parameter defaulting to false, requiring select to have one use. foldOpIntoPhi has logic for multiple identical uses. The fix doesn't handle these constraints properly.
- **Expected Issue**: Incorrect transformation when select/phi has multiple uses that aren't identical shufflevectors.

### Strategy 5: Shuffle mask with poison elements

- **Target**: Both transformations
- **Rationale**: Shuffle masks with poison elements could interact badly with select/phi semantics. The transformation might incorrectly propagate poison or change poison semantics.
- **Expected Issue**: Incorrect poison semantics when shuffle mask contains poison elements.

### Strategy 6: Type mismatch in phi transformation

- **Target**: shufflevector(phi(x, y), C) transformation
- **Rationale**: The phi might have incoming values of different types that get shuffled differently. The transformation assumes uniform shuffling across all incoming values.
- **Expected Issue**: Type safety violation or miscompilation when phi incoming values have different vector types or sizes.

### Strategy 7: Constant RHS but non-constant shuffle mask

- **Target**: Both transformations
- **Rationale**: The fix checks RHS is constant but doesn't verify the shuffle mask is constant. Non-constant shuffle masks could have different semantics when distributed.
- **Expected Issue**: Incorrect transformation with dynamic shuffle masks.

## Agent Review

### Bug Triggering Analysis
This test case triggers the "Shuffle mask with poison elements" strategy. The shuffle mask contains a poison element (index poison). When the fix transforms `shufflevector(select(c, x, y), poison, mask)` to `select(c, shufflevector(x, poison, mask), shufflevector(y, poison, mask))`, it must preserve poison semantics correctly. The transformation appears correct according to alive2, but there could be subtle issues with poison propagation that aren't caught by the verification.

### Fix Weakness Analysis
The fix doesn't have special handling for poison elements in shuffle masks when distributing across select/phi. While the transformation is mathematically correct (poison in mask produces poison in result regardless of select condition), there might be edge cases where the interaction between poison in mask and other optimizations could cause issues. The fix relies on `FoldOpIntoSelect` and `foldOpIntoPhi` which may not handle all poison semantics correctly for vector operations.
