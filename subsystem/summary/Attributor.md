# Subsystem Knowledge for Attributor

## Elements Frequently Missed

*   **Stack-Allocated Temporary Query Objects**: The optimization pass frequently mishandles the lifetime of short-lived, stack-allocated objects used for immediate analysis queries. It fails to distinguish these from long-lived heap objects when managing cache persistence.
*   **Cleanup Triggers during State Updates**: The logic misses necessary cleanup triggers for temporary objects when the system is in a specific state (e.g., "update in progress"). A coarse-grained state flag incorrectly suppresses the removal of temporary data.
*   **Recursive Control Flow Dependencies**: The analysis struggles to correctly manage the scope of queries generated by recursive function calls. These patterns necessitate sub-queries that, if mishandled during a parent query's update phase, lead to memory corruption.

## Patterns Not Well Handled

### Pattern 1: Nested Reachability Queries During Cache Iteration
This pattern involves scenarios where the analysis engine iterates over a set of existing, long-lived reachability queries to update their state. During this iteration, the re-evaluation of a specific query requires generating a new, temporary sub-query (often allocated on the stack). The optimization pass handles this poorly because the mechanism designed to protect the long-lived queries during iteration (a state flag) inadvertently protects the temporary sub-query as well. This prevents the temporary query from being removed from the cache when its scope ends, resulting in a dangling pointer to a destroyed stack frame.

### Pattern 2: Inter-procedural Recursion with Dynamic Cache Updates
The optimization pass shows weakness in handling inter-procedural recursion (e.g., a function calling itself or a cycle in the call graph) combined with dynamic caching. In the provided example, `recursive_reachability` creates a dependency cycle. Resolving this requires the reachability analysis to perform nested checks. The pattern of updating the cache while simultaneously resolving recursive dependencies exposes flaws in the lifetime management of the analysis data structures, specifically when temporary results are mixed with persistent cache entries.
